# Phase 1: 주장 단위 분해 프롬프트

## 원칙

> 4대 불변 원칙, 판정 체계, 근거유형(evidence_type), claim_type 분류의 전체 정의는 **CLAUDE.md** 참조.
> 아래는 Phase 1 Claim 추출 전용 규칙만 기술한다.

---

아래 문서에서 **검증 가치가 있는 핵심 주장(claim)만 추출**하라.

## 추출 기준

포함:
- 구체적 수치가 포함된 문장 (매출, PER, 목표가, TVL, GDP, 금리, CPI 등)
- 인과 관계를 주장하는 문장 ("A 때문에 B가 될 것")
- 예측/전망 문장 ("~할 것으로 전망")
- 리스크/기회를 단정하는 문장
- 법적 의무/권리를 규정하는 조항 (계약서)
- 매크로 지표 전망 (금리 경로, GDP 성장률, 인플레이션 전망, 통화정책 방향)
- 지정학 시나리오 판단 (Base/Bull/Bear 확률, 에너지 경로, 제재 영향 전망)

제외:
- 일반적 배경 설명 ("삼성전자는 반도체 기업이다")
- 출처가 명시된 직접 인용
- 단순 정의/용어 설명

## 각 claim에 부여할 정보

```json
{
  "claim_id": "c001",
  "text": "원문 그대로 인용",
  "claim_type": "수치주장 | 인과주장 | 예측 | 사실진술 | 의견 | 조항",
  "evidence_type": "fact | estimate | opinion",
  "location": "페이지, 문단, 섹션 등 위치",
  "depends_on": ["이 claim이 의존하는 다른 claim의 ID"]
}
```

## claim_type 분류 기준

| 유형 | 기준 | 예시 |
|---|---|---|
| 수치주장 | 구체적 수치 포함 | "매출 10조원", "PER 15배", "TVL $5B" |
| 인과주장 | A→B 관계 주장 | "금리 인상으로 부동산이 하락할 것" |
| 예측 | 미래 전망 | "하반기 반등 전망", "2027년까지 성장" |
| 사실진술 | 과거/현재 사실 | "지난달 발표된", "현재 시행 중인" |
| 의견 | 주관적 판단 | "매력적인 밸류에이션", "긍정적 전망" |
| 조항 | 법적 의무/권리 | "갑은 을에게 30일 이내 통지해야 한다" |

## ★ 근거유형(evidence_type) 분류 기준

> CFA 표준 "Distinguish between fact and opinion" 명시 의무.
> 동일 수치도 추정이면 검증 기준이 달라야 한다.

| 근거유형 | 기준 | 예시 | 판정 연동 |
|---|---|---|---|
| `fact` | 확인된 과거/현재 데이터 | "2025년 매출 10조원", "현재 PER 15배" | 기본 판정 기준 적용 |
| `estimate` | 미래 추정·전망·예측 수치 | "2027년 매출 15조 전망", "금리 3.0% 예상" | ① Fact 판정 기본값 🟡 하향 |
| `opinion` | 정성적 판단, 주관적 평가 | "매력적인 밸류에이션", "성장세가 견고하다" | ⑤ Incentive 맥락에서 참조 |

### claim_type × evidence_type 매트릭스 (전 조합)

```
수치주장 | fact     → ① Fact 우선 검증
수치주장 | estimate → ① Fact + 🟡 하향 기본값 (추정치이므로 🟢 상한 제한)
수치주장 | opinion  → 해당 없음 (수치가 있으면 fact 또는 estimate. opinion 불가)

인과주장 | fact     → ③ Logic + ① Fact (인과 근거가 검증된 데이터 기반)
인과주장 | estimate → ③ Logic + ④ Temporal (인과 근거가 전망치 기반)
인과주장 | opinion  → ③ Logic + ⑥ Omission 우선 검증

예측     | fact     → 해당 없음 (예측은 본질적으로 estimate)
예측     | estimate → ④ Temporal + ③ Logic (KC 추출) ← 기본 조합
예측     | opinion  → ③ Logic + ⑤ Incentive (정성적 전망)

사실진술 | fact     → ① Fact 검증 ← 기본 조합
사실진술 | estimate → 모순 (사실진술인데 추정이면 → claim_type을 '예측'으로 재분류)
사실진술 | opinion  → 모순 (사실진술인데 의견이면 → claim_type을 '의견'으로 재분류)

의견     | fact     → 해당 없음 (데이터 기반이면 의견이 아님)
의견     | estimate → 해당 없음 (수치 추정이면 의견이 아님)
의견     | opinion  → ⑤ Incentive 맥락에서 참조 ← 기본 조합

조항     | fact     → ② Norm 검증 (법률 계약) ← 기본 조합
조항     | estimate → 해당 없음 (법적 조항은 사실)
조항     | opinion  → 해당 없음 (법적 조항은 사실)
```

★ **"해당 없음" 또는 "모순" 조합**을 만나면: claim_type을 재분류하라. evidence_type을 억지로 맞추지 말 것.

## ★ 복합 claim 분해 규칙

하나의 문장에 여러 유형이 섞여 있으면 **원자 단위로 분해**하라.

### 분해 예시

원문: "반도체 재고조정이 완료(사실진술)되어 하반기 매출 15조원 달성이 가능(수치주장+예측)할 것"

분해:
- c005: "반도체 재고조정이 완료" → 사실진술 | fact (depends_on: [])
- c006: "하반기 매출 15조원 달성이 가능" → 예측 | estimate (depends_on: ["c005"])
- c007: "매출 15조원" → 수치주장 | estimate (depends_on: ["c006"])

### 분해 원칙

1. **수치는 항상 별도 claim으로 분리.** 수치가 포함된 인과주장에서 수치 부분을 수치주장으로 빼라.
2. **전제→결론 구조가 보이면 분리.** 전제를 먼저 등록하고, 결론의 depends_on에 전제 ID를 넣어라.
3. **하나의 claim에 2개 이상의 검증 방법이 필요하면 분해 대상이다.**

## ★ 우선순위 선별 기준

10~20개 사이로 추출하되, 아래 우선순위로 선별하라:

### 반드시 포함 (High Priority)
- 문서의 **핵심 결론을 지탱하는** claim (목표가, 투자의견, 최종 판단의 근거)
- **다른 claim이 의존하는** 전제 claim (이것이 틀리면 연쇄적으로 무너짐)
- **검증 가능한 수치주장** (MCP로 1차 소스 조회 가능)
- **반복적으로 등장하는** 핵심 논리 (문서의 뼈대)

### 가능하면 포함 (Medium Priority)
- 리스크 평가와 직결된 인과주장
- 경쟁사·시장 비교 수치
- 밸류에이션 모델의 핵심 가정

### 제외 가능 (Low Priority)
- 부수적 통계 (시장 점유율 추이 중 비핵심 연도)
- 문서의 결론에 영향을 미치지 않는 배경 수치
- 이미 포함된 claim으로부터 자동 도출되는 수치 (예: 합계)

## ★ 의존성(depends_on) 매핑 규칙

단순히 "관련 있다"가 아니라, **이것이 틀리면 저것도 무너지는** 관계만 연결하라.

| 관계 유형 | depends_on 연결 여부 | 예시 |
|---|---|---|
| 전제→결론 | ✅ 연결 | "재고조정 완료"(c005) → "매출 반등"(c006) |
| 수치→인과주장 | ✅ 연결 | "PER 15배"(c001) → "저평가"(c002) |
| 독립적 병렬 | ❌ 미연결 | "매출 10조"(c003)와 "R&D 비용 1조"(c004) |
| 같은 근거, 다른 결론 | ❌ 미연결 | "금리 3%"에서 "부동산↓"과 "은행수익↑" |

---

## 규칙

1. 10~20개 사이로 추출. 20개 초과 시 우선순위 기준으로 선별.
2. depends_on: 인과 의존 관계만 연결. 단순 관련성은 연결하지 않는다.
3. 위치를 최대한 구체적으로 (§3, 페이지 5, "리스크 요인" 섹션 등)
4. 원문을 그대로 인용. 요약하거나 바꾸지 말 것.
5. 복합 claim은 반드시 분해. 하나의 claim = 하나의 검증 단위.

## 출력

MCP 도구 `verify_add_claim()`을 각 claim마다 호출하라:

```
verify_add_claim("c001", "Fed Funds Rate 3.64%", "수치주장", evidence_type="fact", location="§3")
verify_add_claim("c002", "2027년 금리 3.0% 전망", "수치주장", evidence_type="estimate", location="§3")
verify_add_claim("c003", "금리 인하로 실적 반등 전망", "인과주장", evidence_type="opinion", location="Summary", depends_on=["c002"])
...
```
