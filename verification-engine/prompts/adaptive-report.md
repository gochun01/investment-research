# Phase 5 확장: 자율 판단 보고서 생성

> 고정 구조가 아닌, 검증 결과의 성격이 보고서 형태를 결정한다.
> 기존 Phase 5(7+1 섹션)를 대체하거나 선택적으로 사용.

---

## Phase 5-A: 데이터 읽기 (result_json에서 5가지 추출)

```
① 핵심 주장 (Core Claim)
   "이 검증이 가장 강하게 말하고 있는 것 1문장"
   추출: direction_rationale 또는 critical_flags에서

② 긴장 (Tension)
   "검증 결과 안에서 충돌하는 힘이 있는가?"
   추출: 🟢 claim과 🟡/🔴 claim의 대비. 층별 판정 불일치.

③ 무게 중심 (Gravity)
   "데이터의 양과 깊이가 어디에 집중되어 있는가?"
   추출: claim 수가 많은 층, evidence가 풍부한 층, 🟡/🔴가 집중된 층

④ 시간축 (Timeline)
   "과거 회고인가, 현재 상태인가, 미래 분기인가?"
   추출: temporal 판정, invalidation_triggers, valid_until

⑤ 미해결 (Unresolved)
   "검증이 답하지 못한 질문이 있는가?"
   추출: kc_extracted, critical_flags 중 미해소
```

## Phase 5-B: 보고서 유형 자동 판정

```
유형 A — 대립형: Tension 명확. 🟢 vs 🟡/🔴 대비.
  구조: 확인된 것 vs 미확인된 것 → 분기 조건
  예: 천궁 검증 — 주가 🟢 vs 요격률·PSR 🟡

유형 B — 서사형: Tension 없이 전체가 한 방향.
  구조: 배경 → 전개 → 판정 → 권고
  예: 전체 🟢 (경미한 지적만)

유형 C — 스냅샷형: 정기 검증 업데이트.
  구조: 현재 판정 한눈에 → 이전 대비 delta → 주의 사항
  예: 동일 문서 재검증

유형 D — 분기형: invalidation_triggers가 핵심.
  구조: 현재 판정 → 트리거별 시나리오 → 각 경로별 영향
  예: FOMC 전 검증 — 금리 인상/동결 시나리오

유형 E — 복합형: 위 2개+ 동시 해당.
  구조: 주 유형 + 보조 유형 조합.
```

## Phase 5-C: 섹션 자율 구성

```
규칙:
├── 무게 중심이 가장 큰 영역 → 가장 큰 섹션
├── 무게 중심이 중간 → 보조 섹션
├── 데이터가 없는 영역 → 만들지 않는다
├── Unresolved가 있으면 → 마지막에 Open Questions
└── 섹션 수: 데이터가 결정. 3개일 수도, 8개일 수도 있다.
```

## Phase 5-D: 렌더링 불변 규칙

```
디자인:
├── CSS 변수 사용 (다크/라이트 전환)
├── 폰트: Libre Baskerville(제목) + DM Sans(본문) + DM Mono(수치)
├── 단일 .html (외부 의존 = Google Fonts만)
├── topbar + 테마 토글 항상 포함
└── 반응형 + 인쇄 대응

콘텐츠:
├── 첫 화면에 Core Claim 1문장
├── 모든 수치에 출처 표기
├── 판정(🟢🟡🔴)을 시각적으로 표현
├── 면책 조항 하단 포함
└── 데이터 기준 시점 명시

금지:
├── 데이터 없는 섹션 생성
├── 모든 섹션 균등 크기
├── JSON 구조 그대로 나열
└── "추가 분석이 필요합니다" 같은 빈 문장
```

## Phase 5-E: 글쓰기 원칙

```
1. 첫 문장이 결론이다 (배경부터 시작 금지)
2. 수치가 먼저, 해석이 뒤에
3. 확신과 추정을 구분한다
4. 빈 공간은 미덕이다 (데이터 3줄이면 3줄로 끝)
5. 독자는 바쁘다 (스크롤 최소화)
```

## Phase 5-F: 보고서 자기 검증

```
V1. Core Claim이 본문과 일치하는가?
V2. 핵심 수치가 빠짐없이 반영되었는가?
V3. 데이터 없이 만들어진 섹션이 있는가?
V4. 데이터 양과 섹션 크기가 비례하는가?
V5. 첫 화면만으로 핵심 파악 가능한가?
```

## report-class 자동 판정

```
critical_flags 있고 🔴 다수 → CRISIS ALERT (빨강)
invalidation_triggers가 핵심 → SPECIAL REPORT (진한 파랑)
정기 재검증 성격 → RESEARCH NOTE (네이비)
전략 변경 권고 포함 → STRATEGY UPDATE (골드)
기본값: RESEARCH NOTE
```

## 기존 Phase 5(7+1 섹션)와의 관계

```
기존 Phase 5: 7+1 섹션 고정. topbar+float-nav+SHORT/FULL 토글.
  → "완벽한 정형 검증 보고서"가 필요할 때.

자율 Phase 5: 섹션 수/구조가 데이터에 따라 유동.
  → "이 검증 결과로 최적의 보고서 알아서"일 때.

선택 기준:
  사용자가 "보고서 만들어" → 자율 (이 문서)
  사용자가 "정형 보고서로" 또는 "기존 양식으로" → 기존 Phase 5
  기본값: 자율
```
