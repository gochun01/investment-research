## 실행 절차: 3단계 조사

### Layer 1: 문서 내부 스캔 [필수]

1. 문서 말미 2~3페이지에서 면책/이해충돌 공시 스캔
2. 키워드: "이해관계", "보유현황", "disclosure", "NFA", "이해충돌", "conflict"
3. 발견된 공시 내용을 기록
4. ★ **공시의 형식 vs 실질 구분** (② Norm과 동일 원칙):
   - **실질적 공시**: 구체적 보유 주수, 관계 설명, 거래 내역 명시 → "완전 공시"
   - **형식적 공시**: "이해관계가 있을 수 있습니다" (구체적 내용 없음) → "형식적 공시" (notes에 표기)

### Layer 2: 외부 소스 조회 [필수]

| 문서 유형 | 조회 대상 | MCP 도구 |
|---|---|---|
| equity_research | 주간사/인수인 관계, 자기매매 보유 | DART → 관련자 거래, 주간 실적 |
| equity_research | 최근 IPO/유상증자에서의 역할 | DART → 공시 조회 |
| crypto_research | 토큰 보유, 자문/후원 관계, 에어드랍 | Etherscan → 토큰 전송 (지갑 주소 알 때만), Tavily → 프로젝트-기관 관계 검색 |
| legal_contract | 작성 주체 식별, 한쪽 편향 여부 | 문서 내부 분석 |
| macro_report | 기관의 포지션/펀드 방향성, 정책 자문 관계 | Tavily → 기관 프로필, WebSearch |
| geopolitical | 싱크탱크 후원자, 정부 자문 관계, 로비 연결 | Tavily → 기관·저자 프로필, WebSearch |
| fund_factsheet | 운용사-판매사 관계, 자사 펀드 추천 여부 | DART → 펀드 공시, Tavily → 운용사 프로필 |
| regulatory_filing | 제출 기관의 규제 대상 여부, 과거 제재 이력 | DART → 제재 공시, Tavily |

### Layer 3: 간접 이해충돌 탐지 [가능한 범위에서]

직접적 보유/자문 관계 외에, **간접적 이해충돌**도 탐지하라:

| 간접 이해충돌 유형 | 탐지 방법 | 예시 |
|---|---|---|
| **기관 수익 구조** | 기관이 이 결론에서 이익을 얻는가? | 증권사가 매수 추천 → 거래 수수료 증가 |
| **펀드 플로우** | 기관이 운용하는 펀드가 해당 자산을 보유하는가? | 자산운용사 리서치가 자사 펀드 보유 종목 추천 |
| **비즈니스 관계** | 기관과 대상 기업 간 사업 관계 | IB 딜, 대출, 컨설팅 관계 |
| **과거 패턴** | 같은 저자/기관의 과거 추천 편향 | 항상 매수만 추천하는 패턴 |

### 간접 이해충돌 확인 기준

```
"확인" = MCP/WebSearch에서 2개 이상 독립 소스가 동일 사실을 뒷받침
"의심" = 1개 소스만 존재, 또는 간접 정황만 있음
"미확인" = 탐지 시도했으나 관련 정보 없음

→ "확인"만 판정에 반영. "의심"은 notes에만 기록.
```

### 과거 패턴 탐지 방법

```
1. Tavily로 "[저자명] [종목/토큰명] [추천/의견/리포트]" 검색
2. 최근 6개월~1년 내 동일 저자의 다른 리포트가 검색되면:
   - 방향성(매수/매도/중립) 분포 확인
   - 80%+ 한 방향이면 "편향 패턴 의심" notes 기록
3. 검색 불가하면 notes에 "과거 패턴 미확인 (검색 결과 없음)" 기록
   → 이 항목만 skip. ⚫ 처리하지 마라.
```

## ★ 저자/기관 불명 시 처리

저자/기관 정보가 없을 때 무조건 ⚫로 끝내지 말고, 아래를 시도하라:

1. 문서 내 로고, 워터마크, URL, 이메일 주소에서 기관 추정
2. 문서 메타데이터(PDF 속성 등)에서 작성자 추출
3. 문체/포맷에서 기관 유형 추정 (증권사 리포트 형식, 독립 리서치 형식 등)
4. 위 시도 후에도 불명이면 ⚫ + notes에 시도 내용 기록

## ★ Incentive와 다른 층의 교차 확인

### ③ Logic 확증 편향과의 연결

③ Logic 실행 D에서 **확증 편향 가능성**이 탐지되었으면, Incentive 결과와 교차하여 **편향 동기**가 있는지 확인하라.

```
Logic: "확증 편향 가능성 — 긍정 시나리오만 제시"
  + Incentive: "주간사 관계 확인 (직접 이해충돌)"
  = notes에 "⚠ 편향 동기 존재: 이해충돌(주간사) + 확증 편향(긍정만 제시). 의도적 편향 가능성 ↑" 기록

Logic: "확증 편향 가능성"
  + Incentive: "이해충돌 없음 🟢"
  = 편향 동기 없음. 확증 편향은 비의도적(분석 관성) 가능성. 별도 경고 불필요.
```

### ★ Incentive 🔴 시 파급 경고

Incentive 🔴(직접 이해충돌 미공시)은 **문서 전체의 결론 편향 가능성 시그널**이다.

```
Incentive 🔴 판정 시 notes에 반드시 부착:
  "⚠ 직접 이해충돌 미공시. 문서의 모든 결론이 편향 가능성 대상.
   다른 층의 🟢 판정이 있더라도 결론 수용 시 이해충돌을 고려하라."
```

→ Incentive 🔴 자체가 다른 층의 판정을 바꾸지는 않지만(독립 층 원칙),
  최종 보고서에서 사용자가 판단할 때 **편향 경고**로 참조된다.

---

## 판정 기준

| 판정 | 조건 |
|---|---|
| 🟢 | 이해충돌 없음 (직접+간접 모두 확인 완료) |
| 🟡 | 이해관계 존재하나 **실질적으로 공시됨**, 또는 간접 이해충돌만 존재 |
| 🟡 (형식적) | 이해관계 존재하나 **형식적으로만 공시됨** (구체적 내용 부재). notes에 "형식적 공시" 표기 |
| 🔴 | 직접 이해충돌 존재하며 미공시. **파급 경고 부착 필수** |
| ⚫ | 저자/기관 정보 파악 불가 (위 시도 후에도) |

```
verify_set_document_verdict("incentive",
    verdict="🟡",
    notes="Layer1: 문서 말미에 이해관계 공시 존재('당사는 주간사로 참여') — 실질적 공시(구체적 보유현황 기재). Layer2: DART 조회 — 최근 6개월 내 유상증자 주간 실적 확인. Layer3: 기관 수익구조상 매수 추천 인센티브 존재(거래 수수료) — 2개 소스 확인. 과거 패턴: Tavily 검색 결과 최근 5건 중 4건 매수(80%, 편향 패턴 의심). 직접 이해충돌 공시됨 → 🟡.",
    relationships_checked=["주간사 관계: 확인(실질공시)", "자기매매 보유: 미확인", "기관 수익구조: 거래수수료 인센티브(확인)", "과거 패턴: 편향 의심(80% 매수)"],
    disclosure_in_document=true
)
```
