## 원칙

> 아래는 ⑥ Omission Ground 전용 규칙만 기술한다.

## ⑥ Omission 전용 원칙

- **BBJ Break 0개 🟢 금지** (V-05). 최소 1개의 반론을 생성한 후, 그것이 문서에 언급되어 있어야 🟢.
- **약한 Break로 🟢 달성 금지**. Break는 문서의 핵심 결론을 뒤집을 수 있는 수준이어야 한다. "사소한 리스크"를 Break로 제시하여 형식적 🟢을 달성하지 마라.
- **역할 경계**: Omission은 "이 리스크가 실질적으로 다뤄졌는가?"(내용). ② Norm은 "이 항목이 형식적으로 존재하는가?"(형식). 둘은 독립이다.

## 실행 A: 체크리스트 대조 [필수]

아래 체크리스트의 각 필수 항목이 문서에서 언급되었는지 키워드 스캔하라.

### 체크리스트

{omission_checklist}

> 각 항목은 `{id, item, impact, keywords}` 구조의 JSON 배열이다.
> 상세 스키마는 `schemas/io-contracts.md` 참조.

1. 각 항목의 keywords를 문서에서 검색
2. 발견 → covered
3. 미발견 → missing (임팩트가 critical이면 🔴, high이면 🟡)

### 체크리스트와 BBJ Break의 중복 방지

체크리스트에서 이미 missing으로 플래그된 리스크를 실행 B에서 BBJ Break로 중복 생성하지 마라. BBJ Break는 **체크리스트에 없는 리스크**를 찾는 것이다.

## 실행 B: BBJ Break 생성 — 4관점 전환법 [필수]

### ★ ③ Logic KC 결과 활용

BBJ Break를 만들기 전에, **③ Logic의 kc_extracted를 확인**하라.

```
KC 중 '약화 중' 또는 '미확인(🟡)' 전제 → BBJ Break의 유력한 소재.

예:
  KC-2: "SK하이닉스 점유율 유지" → 🟡 미확인
  → Break 소재: "삼성 HBM3E 수율 개선으로 점유율 역전 가능성"

KC 결과가 {fact_evidence}에 포함되어 있지 않으면, Logic의 출력(notes, kc_extracted)을
대화 컨텍스트에서 참조하라.
```

### ★ BBJ Break 생성의 핵심 문제

LLM이 가장 빠지기 쉬운 함정은 **문서의 논조에 끌려가서 약한 반론만 만드는 것**이다.
강한 Break를 만들려면 **관점을 의도적으로 전환**해야 한다.

### 4관점 전환법

아래 4개 관점에서 **각각 1개씩** 잠재적 Break를 도출하고, 그 중 상위 1~2개를 선택한다.

```
관점 1: 약세론자 (Bear Case)
  "이 문서의 결론이 완전히 틀리려면 무엇이 일어나야 하는가?"
  → 결론의 정반대 시나리오를 MCP 데이터로 뒷받침할 수 있는가?
  예: 문서가 "매수"면 → "이 기업이 -30% 빠지려면?"

관점 2: 규제자/감독자
  "규제 당국이 이 시장/기업/프로토콜에 개입한다면?"
  → 진행 중인 규제 논의, 입법 동향, 과거 규제 선례를 MCP로 조회
  예: SEC 소송, 금감원 검사, FATF 권고

관점 3: 매크로 역풍
  "바깥 환경이 바뀌면 안쪽 결론이 유지되는가?"
  → 금리, 환율, 유동성, 지정학 중 문서가 가정하고 있는 매크로 환경을 식별
  → 그 가정이 깨질 시나리오
  예: "금리 동결 전제"인데 인플레 재점화 시

관점 4: 경쟁/대체
  "경쟁자나 대체재가 이 기회를 빼앗을 수 있는가?"
  → 경쟁사 동향, 대체 기술, 시장 구조 변화
  예: 경쟁 프로토콜의 TVL 급증, 대체 기술 상용화
```

### Break 선택 기준

4개 관점에서 각 1개씩 = 4개 후보. 이 중 **상위 1~2개만 선택**:

| 선택 기준 | 가중치 |
|---|---|
| **임팩트 크기**: 이 Break가 현실화되면 결론이 얼마나 바뀌는가? | 높음 |
| **발생 가능성**: 현재 시그널이 있는가? (MCP로 확인) | 높음 |
| **시간 근접성**: 가까운 시일 내 발생 가능한가? | 중간 |
| **문서의 침묵**: 문서가 이 리스크를 전혀 언급하지 않았는가? | 높음 |

### Break 생성 금지 사항

- 극단적/비현실적 시나리오 금지 ("소행성 충돌", "모든 거래소 파산")
- 문서 주제와 무관한 리스크 금지
- 문서가 이미 다룬 리스크를 Break로 제시하지 않음
- 체크리스트에서 이미 missing으로 플래그된 리스크를 Break로 중복 제시하지 않음
- **3개 이상 선택 금지**. 핵심만 골라라.

### Break 검증 — MCP 시그널 확인 기준

선택된 Break에 대해:

```
1. MCP로 시그널 존재 여부 조회 (단순 상상이 아니라 데이터 뒷받침)

   "시그널 있음" 기준:
   - 구체적 수치 (가격 변동, TVL 변화, 공시 내용) → 강한 시그널
   - 뉴스/공시 2건 이상이 같은 방향 → 강한 시그널
   - 뉴스 1건만 존재, 정량 데이터 없음 → 약한 시그널 (notes에 "약한 시그널" 표기)
   - 일반론만, MCP 데이터 없음 → 시그널 없음 (Break 후보에서 제외)

2. 문서에 해당 리스크가 언급되었는지 확인
   - 미언급 → 생략 플래그
   - 언급됨 → 플래그 아님
```

## 실행 C: 동심원 층위 정합성 [필수]

> 문서는 특정 층위(개별종목/토큰)에 집중하면서 바깥 층위(매크로/규제)를 전제로 깔고 간다.
> 그 전제가 현재도 유효한지 확인하는 것이 동심원 검사다.

### 동심원 층위 구조

```
┌─────────────────────────────────────┐
│           글로벌 매크로              │  ← 가장 바깥
│  ┌───────────────────────────────┐  │
│  │        국가/정책 환경          │  │
│  │  ┌─────────────────────────┐  │  │
│  │  │      섹터/산업           │  │  │
│  │  │  ┌───────────────────┐  │  │  │
│  │  │  │  개별 종목/토큰    │  │  │  │  ← 문서의 초점
│  │  │  └───────────────────┘  │  │  │
│  │  └─────────────────────────┘  │  │
│  └───────────────────────────────┘  │
└─────────────────────────────────────┘
```

### 실행 방법

1. 문서가 다루는 층위를 식별 (대부분 안쪽 1~2층)
2. **문서가 암묵적으로 전제하는 바깥 층위 조건**을 추출
   예: "반도체 매수" → 전제: 경기 회복, AI CAPEX 유지, 대중 수출규제 현상 유지
3. 각 바깥 층위 조건의 **현재 상태를 MCP로 조회** ({fact_evidence}에 있으면 재사용)
4. 바깥 층위가 변하고 있는데 안쪽 결론이 이를 반영하지 않으면 → 플래그

### 바깥 층위 조건 추출 가이드

LLM이 주관적으로 추출하지 않도록 아래 프레임을 사용하라:

| 문서 초점 | 확인할 바깥 층위 | MCP 조회 대상 |
|---|---|---|
| 개별종목 | 섹터 + 매크로 + 정책 | 업종 지수, FRED 금리/PMI, 규제 동향 |
| 섹터 | 매크로 + 정책 | FRED 금리/GDP, 산업 정책 뉴스 |
| 매크로 | 글로벌 매크로 + 지정학 | DXY, 유가, 지정학 뉴스 |
| 토큰/프로토콜 | 크립토 매크로 + 규제 | BTC 도미넌스, 규제 뉴스 |

### 동심원 → finalize 전달

동심원에서 발견된 **"전제 약화/붕괴"는 notes에 기록**한다. finalize가 result_json에서 이 notes를 읽어 validity_condition에 반영한다.

### 동심원 결과 기록

notes에 포함:
```
동심원 검사:
- 매크로 전제: "글로벌 경기 완만한 회복" → FRED PMI 48.2 (수축 영역). 전제 약화 🟡
- 정책 전제: "대중 수출규제 현상 유지" → BIS Entity List 최근 추가 없음. 유지 🟢
- 섹터 전제: "AI CAPEX 증가 지속" → MS/Google CAPEX 가이던스 상향. 유지 🟢
```

## 실행 D: 종합 판정

### 판정 기준

| 판정 | 조건 |
|---|---|
| 🟢 | critical 누락 없음 + BBJ Break 전부 문서에 언급됨 + 동심원 전제 유효 |
| 🟡 | high 항목 누락 또는 BBJ Break 1개 미언급 또는 동심원 전제 약화 |
| 🔴 | critical 항목 누락 또는 BBJ Break 2개 미언급 또는 동심원 전제 붕괴 |
| ⚫ | 해당 섹터 체크리스트 미구축 |

### ★ 문서 전체 판정 + claim별 판정의 관계

```
문서 전체 판정: 체크리스트 + BBJ Break + 동심원 → verify_set_document_verdict
claim별 판정:  claim 고유 반론 → verify_set_verdict

집계 규칙:
- 문서 전체 판정이 최종 Omission 대표 판정
- claim별 판정은 해당 claim의 개별 notes에 기록 (claim의 논증 결함 추적용)
- 문서 🟡 + claim 🔴인 경우 → 문서 전체 판정을 🔴로 격상
  (claim 수준의 심각한 논증 결함이 문서 결론에 영향)
```

## 출력

### 문서 전체 레벨:
```
verify_set_document_verdict("omission",
    verdict="🟡",
    notes="체크리스트: om_fin_004 missing(high). BBJ Break 1건 미언급. 동심원: 매크로 전제 약화(PMI 48.2). KC 소재 활용: KC-2(점유율) 미확인 → Break 생성에 반영.",
    checklist_matched=["om_fin_001", "om_fin_005"],
    checklist_missed=["om_fin_004"],
    bbj_breaks=[
        {"break_text": "SEC/FSOC 규제 강화 가능성 (관점2: 규제자)", "in_document": false, "verdict": "🔴 미언급. MCP: SEC 의장 발언 2건 확인(강한 시그널)"},
        {"break_text": "디스트레스드 펀드 저가 매수로 시장 자정 (관점1: 약세론자)", "in_document": true, "verdict": "언급됨"}
    ]
)
```

### ★ claim별 개별 판정 [필수]

applicable_layers에서 omission=True인 claim(인과주장, 예측)에 대해 **claim 특유의 반론을 별도 생성**하라.

**문서 전체 BBJ Break를 그대로 복사하지 마라.** 각 claim의 인과 체인 또는 예측 논리에 고유한 반론을 만들어야 한다.

```
절차:
1. claim의 핵심 인과 체인을 식별: "A → B → C"
2. 이 체인을 깨뜨릴 수 있는 지점을 찾는다:
   - A가 거짓일 경우? (전제 붕괴 — ③ Logic KC와 연동)
   - A→B 연결이 끊어질 경우? (인과 단절 — Bridging Break)
   - B→C 대신 B→D가 될 경우? (경로 분기 — Jumping Break)
3. 해당 반론이 문서에 언급되었는지 확인
4. claim별로 verify_set_verdict 호출
```

예:
```
claim: "금리 인하로 부동산 반등 전망" (인과주장)
  인과 체인: 금리 인하 → 대출 부담 감소 → 수요 회복 → 가격 반등
  claim 고유 반론: "대출 규제(DSR) 강화로 금리 인하에도 대출 접근 제한"
    → 인과 단절 지점: 금리 인하 → (DSR 차단) → 수요 미회복
    → 문서 언급 여부: 미언급 → 🟡

verify_set_verdict("c003", "omission",
    verdict="🟡",
    notes="claim 고유 반론: DSR 규제가 금리 인하→수요 회복 인과 단절 가능성. 문서 미언급. MCP: 금감원 DSR 강화 뉴스 1건(약한 시그널).",
    bbj_breaks=[{"break_text": "DSR 규제 강화로 금리 인하→수요 회복 인과 단절", "in_document": false, "verdict": "🟡 미언급(약한 시그널)"}]
)
```
