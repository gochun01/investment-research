4. Connection Map (파일 간 참조)

4-1. 코드 의존성 (Python import)

mcp_server.py → core/engine.py (VerificationEngine) → core/layers/ (NormLayer, LogicLayer, OmissionLayer) core/engine.py → core/models.py (Claim, LayerVerdict, VerificationResult) → core/layers/__init__.py (6개 Layer 클래스) → data/claim_type_matrix.json core/layers/*.py (6개 모두) → core/models.py (LayerVerdict) → data/checklists.json (norm.py, omission.py) → data/rules.json (logic.py)

4-2. 프롬프트 참조 (verify_get_prompt)

mcp_server.py verify_get_prompt(phase) → prompts/{phase}.md (1차 탐색) → prompts/schemas/{phase}.md (2차 탐색) verify_orchestrator() → CLAUDE.md + SKILL.md (결합 반환)

4-3. 플레이스홀더 주입 (mcp_server.py → prompts)

프롬프트 로드 시 엔진 데이터를 자동 주입: {mcp_sources} ← engine.get_fact_sources() → fact_check.md {checklist} ← engine.get_norm_checklist() → norm_check.md {rules} ← engine.get_logic_rules() → logic_check.md {omission_checklist} ← engine.get_omission_checklist() → omission_check.md {check_items} ← engine.get_incentive_checks() → incentive_check.md {missing_items} ← engine.check_coverage() → coverage_recheck.md {fact_evidence} ← Phase 2① Fact 결과 → logic_check.md, temporal_check.md, omission_check.md {claims} ← engine.result.claims → logic_check.md, temporal_check.md {result_json} ← engine.finalize() → self_audit.md {coverage_report} ← engine.get_coverage_report() → self_audit.md {data} ← engine.get_result_dict() → SKILL.md Phase 5 {document} ← 대화 컨텍스트 → extract_claims.md

4-4. 프롬프트 원칙 참조 (CLAUDE.md ← 모든 Phase)

CLAUDE.md (불변 원칙 + 판정 체계 + Tier + KC/BBJ) ← extract_claims.md "CLAUDE.md 참조" ← fact_check.md "CLAUDE.md 참조" ← norm_check.md "CLAUDE.md 참조" ← logic_check.md "CLAUDE.md 참조" ← temporal_check.md "CLAUDE.md 참조" ← incentive_check.md "CLAUDE.md 참조" ← omission_check.md "CLAUDE.md 참조" ← coverage_recheck.md "CLAUDE.md 참조" ← finalize.md "CLAUDE.md 참조"

4-5. 층간 데이터 전달

Phase 0 verify_start() 반환값 ↓ mcp_sources, checklist, rules 자동 주입 Phase 2① Fact evidence ↓ {fact_evidence} 플레이스홀더 → Phase 2③ Logic (KC 확인 시 MCP 재호출 방지) → Phase 2④ Temporal (시점 비교 데이터 재사용) → Phase 2⑥ Omission (BBJ 시그널 확인) Phase 2③ Logic KC ↓ 대화 컨텍스트 → Phase 2⑥ Omission (BBJ Break 소재) → Phase 3+4 finalize (validity_condition) Phase 2⑥ Omission 동심원 ↓ notes → Phase 3+4 finalize (validity_condition 보강) Phase 3+4 result_json ↓ {result_json}, {coverage_report} → Phase 4.5 self_audit ↓ {data} → Phase 5 HTML 보고서

4-6. HTML 보고서 생성 경로

Phase 5 (SKILL.md §5) ↓ Finding 생성 (9필드 × N건) ↓ 섹션 구조 (7+1) ↓ CSS 참조 ← schemas/component-catalog.md HTML 골격 ← output/template-verification.html ↓ [slug]-verification-report.html → output/ 또는 사용자 지정 경로
