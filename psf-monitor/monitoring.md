# PSF 적응형 관찰 프로토콜

> **버전**: v2.0 | **작성일**: 2026-03-23
> **이전**: v1.0은 고정 4중 루프 템플릿이었다. 매일 같은 순서, 같은 형식.
> **전환**: 관찰의 원칙과 리듬만 정의한다. 과정과 깊이는 데이터가 결정한다.

---

## 관찰 원칙

```
1. 튀는 것부터 판다
   전체를 훑고, 가장 크게 변한 것부터 깊이 들어간다.
   고정 순서(Step 1→2→3)가 아니라 데이터가 순서를 정한다.

2. 평소와 다른 것을 먼저 본다
   "같은 것"은 확인만 하고 넘어간다.
   "다른 것"에 시간을 쓴다.

3. 감속·훼손을 가속보다 먼저 본다
   좋은 소식은 놓쳐도 괜찮다. 나쁜 소식을 놓치면 위험하다.
   확인 편향 방어.

4. 설명 안 되는 것을 버리지 않는다
   분류 불가 = 새 구조의 전조일 수 있다. 기록한다.

5. 루프를 건너뛰지 않되, 루프 안의 과정은 자유롭다
   일일/주간/분기의 리듬은 유지한다 (신호 누적을 위해).
   리듬 안에서 무엇을 얼마나 깊이 볼지는 그날의 데이터가 결정한다.
```

## 일일 관찰 (Daily)

```
트리거: "브리핑", "오늘 시장", "아침 점검", "PSF 갱신"
소요: 15~30분 (데이터 상황에 따라 변동)

해야 할 것:
  1. 데이터 수집 — Macro latest.json 읽기 + 맥락에 맞는 MCP 동원
  2. axis-map.md 프로토콜 실행 (1~5단계)
     → 5단계 모두 매일 할 필요 없음. 변화가 없으면 "변화 없음" 기록.
     → 큰 변화가 있으면 깊이 판다.
  3. P층 독립 수집 — 각 Property별 맥락 기반 탐색
     → 쿼리는 고정이 아님. 직전 관측에서 파생.
     → P3이 🔴였으면 "48시간 이후 호르무즈" 같은 상황 쿼리.
  4. 한 줄 판단 기록 — "오늘 이것만 기억한다"

산출물: 형식은 내용이 결정. 변화가 없는 날은 3줄로 끝나도 된다.
```

## 주간 관찰 (Weekly) — 매주 금요일 또는 주말

```
트리거: "주간 리뷰", "축 리뷰", "W Loop"
소요: 15~30분

해야 할 것:
  1. 이번 주 일일 관찰을 되돌아본다
     → "이번 주에 가장 중요했던 변화는?"
     → "누적하면 패턴이 보이는 것이 있는가?"

  2. 축↔PSF 정합성을 점검한다

     정합 매트릭스:
                PSF 국면
              🟢 정상   🟡 경계   🔴 위기
     축 가속    정상     감시     ⚡ 역설
     축 중립    정상     감시     감시
     축 감속    감시     ⚡ 주의   ⚠️ 경보

     ⚡ 역설 (축 가속 + PSF 위기):
       축은 강한데 시장이 나쁘다. 판이 축을 막고 있는 상태.
       "판이 축을 꺾는 것인지, 일시적으로 막는 것인지" 구분 필요.
     ⚠️ 경보 (축 감속 + PSF 위기):
       축도 약해지고 시장도 나쁘다. 축 훼손 긴급 점검.

     이 매트릭스는 macro↔PSF 정합(interfaces-macro.md)과 별개.
       macro↔PSF = 돈의 방향 vs 시스템 스트레스
       축↔PSF = 메가트렌드 방향 vs 시장 상태

  3. 이번 주 해소된 질문과 새로 생긴 질문을 정리한다
     → 지난주 질문 → 이번 주 답(또는 미해소)
     → 다음 주에 답을 구해야 할 질문 1~3개

  4. [미분류] 신호가 누적되고 있는지 확인한다

  5. errors.md 신규 오류를 리뷰한다
     → 이번 주 추가된 ERR-xxx가 있는가?
     → 기존 오류가 반복 발생했는가? → severity 승격
     → 방어 규칙 추가 필요 여부 판단
     → 체크리스트에 항목 추가할 것이 있는가?

산출물: 주간 요약 + 질문 목록 + 오류 리뷰. 길이는 변화의 크기에 비례.
```

## 월간 관찰 (Monthly) — 매월 첫째 주

```
트리거: "월간 점검", "배관 점검", "M Loop"
소요: 30~45분

해야 할 것:
  1. 배관(구조) 변화 — "병목이 이동했는가?"
     → 각 축의 핵심 배관에서 변화가 있었는지 점검
     → 병목 이동 = 가치 포착점 이동

  2. 레짐(⑨) 환경 — "게임판의 중력이 바뀌었는가?"
     → 부채, 국채 시장, Fed 독립성, 텀프리미엄

  3. KC 유효성 — "감시하고 있는 위험이 여전히 유효한가?"
     → 새 위험 발생? 기존 위험 소멸? 임계값 재보정 필요?

  4. 사용하지 않은 데이터 소스 점검
     → "이번 달에 DeFiLlama/CoinMetrics/Dune을 써봤으면 뭘 알 수 있었나?"

산출물: 월간 구조 변화 요약. 변화 없으면 짧게.
```

## 분기 관찰 (Quarterly) — 1·4·7·10월

```
트리거: "분기 진단", "축 진단", "Q Loop"
소요: 60~90분

해야 할 것:
  1. 축 건강 진단 — 3대 조건(비가역/관통/자기강화) 재검증
     → 확정 축(①②③④) 각각에 대해
     → "이 분기에 되돌리려는 시도가 있었나? 성공했나?"
     → "새로 관통한 영역이 있나? 이탈한 영역이 있나?"
     → "루프가 여전히 돌고 있나? 속도가 바뀌었나?"

  2. 레짐 시프트(⑨) 재평가

  3. 축 후보 승격 심사 (⑤⑦ + 새 후보)

  4. ★ 구조 자체의 진화 [핵심]
     → "분기 동안 [미분류] 신호가 패턴을 이루는 것이 있는가?"
     → "새로운 Property, Link, 축 후보가 필요한가?"
     → "기존 구조(ontology)에서 더 이상 유효하지 않은 것이 있는가?"
     → 이 단계에서 ontology.md, axis.md 개정 가능.

  5. axis.md 문서 갱신

산출물: 분기 진단 보고서 + 문서 갱신 내역.
```

## KC 국면 보정 (참고선)

```
국면별로 위험 민감도를 조절하는 참고 기준.
이것은 규칙이 아니라 참고선이다 — 맥락이 최종 판단을 결정한다.

🟢 정상: 기준값 그대로
🟡 경계: 기준값의 ~80% 수준에서 주의 시작
🔴 위기: 기준값의 ~60% 수준에서 주의 시작 + 추가 감시 항목 활성

국면 악화 시: 즉시 민감도 상향.
국면 개선 시: 3일 유예 후 이완. "거짓 평화" 방어.
```

## 루프 간 전달 규칙

### 상향 전달 — 신호가 올라간다

```
D→W: 매일의 observations[]와 unclassified[]가
      state.json accumulation.weekly에 자동 누적.
      축별 신호 카운트(accelerate/decelerate/damage) 갱신.

W→M: 주간 신호 net 방향(+/-)이 월간 추세로 집계.
      주간 미해소 질문이 월간 감시점으로 승격.
      정합 ⚡역설 또는 ⚠️경보 발생 시 월간에서 긴급 점검.

M→Q: 월간 배관 변화가 분기 축 진단의 입력.
      월간 KC 재보정이 분기 전면 검토의 기초.
      월간 레짐 추세가 분기 재평가의 근거.
```

### 하향 전달 — 기준이 내려온다

```
Q→D: 분기 축 건강 판정 → 일일 관측의 감시 강도 조정.
      새 KC/만료 KC → 일일 KC 거리 측정 대상 갱신.
      축 후보 승격 → 일일 관측 범위 확대.
      ★ 구조 진화: ontology.md, axis.md 개정 가능.

M→W: 배관 변화 → 주간 정합 매트릭스의 "축 방향" 갱신.
      KC 재보정 → 주간 트리거 거리 기준 갱신.

W→D: 주간 질문 → 일일 탐색에서 해당 데이터 우선 수집.
      정합 경보 → 일일 태깅에서 해당 축 감시 강화.
```

### 이력 보존 규칙

```
주간 리셋 시: accumulation.weekly → history/YYYY-W##-summary.json 저장 후 리셋.
월간 리셋 시: accumulation.monthly → history/YYYY-MM-summary.json 저장 후 리셋.

보존 기간:
  일일 스냅샷: 30일
  주간 요약: 52주 (1년)
  월간 요약: 12개월
  분기 진단: 영구 보존
```
