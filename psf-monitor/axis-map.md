# PSF 신호 발견 프로토콜

> **버전**: v2.0 | **작성일**: 2026-03-23
> **이전**: v1.0은 고정 매핑 테이블이었다. 답이 정해져 있었다.
> **전환**: 이 문서는 질문만 정의한다. 답은 데이터가 결정한다.

---

## 원칙

```
1. 데이터가 과정을 결정한다
   고정 순서가 아니라, 가장 크게 변한 것부터 판다.

2. 답을 미리 정하지 않는다
   "SOXX +5% = ①AI 🔺" 같은 고정 규칙은 없다.
   같은 +5%라도 맥락에 따라 의미가 다르다.

3. 설명 안 되는 것이 가장 중요하다
   기존 틀로 분류되는 신호보다,
   분류 안 되는 신호가 구조 변화의 전조일 수 있다.

4. 사용 가능한 모든 눈을 쓴다
   FRED+CoinGecko+Tavily만이 아니라,
   등록된 모든 MCP 소스에서 맥락에 맞는 것을 동원한다.
```

## 1단계: 변화 감지 — "뭐가 달라졌나?"

```
수집된 모든 데이터를 직전 관측(state.json)과 비교한다.

질문:
  ① 가장 크게 변한 지표 상위 3개는 무엇인가?
     → 절대값이 아니라 "평소 변동 대비 이탈 정도"로 순위를 매긴다.
     → 평소 범위 = 직전 관측의 맥락에서 판단.

  ② 변하지 않았어야 할 것이 변했는가?
     → 안정적이던 지표의 갑작스런 이동이 가장 강한 신호.

  ③ 변했어야 할 것이 안 변했는가?
     → 큰 이벤트가 있는데 무반응인 지표 = 지연 전파 또는 구조 변화.

  ④ 평소 함께 움직이던 것들이 따로 움직이는가?
     → 상관관계 붕괴 = 새로운 힘이 작용하고 있다는 신호.

  ⑤ 데이터 신선도는 정직한가? ★ 개선 4
     → 각 데이터의 실제 날짜를 확인한다.
     → 당일~2일: [fresh] — 판정에 사용 가능.
     → 3~5일: [recent] — 판정에 사용 가능하나 "[recent 기반]" 태그 필수.
     → 6일+: [stale] — 판정에 사용 시 "[stale 기반]" 태그 + 경고.
     → 추정값은 반드시 "[추정]" 표기. 추정값과 실측값의 차이로 판정하지 않는다.
     → "처음 수집한 값"은 "변화"가 아니라 "최초 관측"으로 구분한다.

이 단계의 산출물:
  "오늘의 주목 변화 목록" — 개수와 내용은 데이터가 결정한다.
  각 변화에 신선도 태그를 부착한다.
  변화가 없으면 "유의미한 변화 없음"도 유효한 산출물이다.
```

## 2단계: 인과 추적 — "어디서 왔고, 어디로 가는가?"

```
1단계에서 발견한 각 변화에 대해:

질문:
  ① 원인은 무엇인가?
     → 정책 결정? 지정학 이벤트? 시장 내부 역학? 기술 변화?
     → 원인이 P(판)에 있으면 → S(구조)→F(흐름)으로 전파 가능성.
     → 원인이 F(흐름)에 있으면 → 역방향(L7/L8) 경로 점검.

  ② 이 변화는 어떤 PSF Property에 닿는가?
     → 고정 매핑이 아니라, 이 변화의 성격에서 추론한다.
     → 하나의 변화가 여러 Property에 동시에 닿을 수 있다.

  ③ 전파 경로에 있는 Link가 있는가?
     → ontology.md의 Link 정본(L1~L8) 중 관련된 것을 확인.
     → 기존 Link로 설명 안 되는 전파 경로가 보이면 기록.

  ④ 이 전파는 얼마나 빠른가?
     → 즉시(시장 반응)인가, 지연(정책→실물)인가?
     → 지연 전파는 "아직 안 왔지만 올 것" — 선제 감시 대상.

이 단계의 산출물:
  각 주목 변화별 "원인 → 경로 → 도착점" 서술.
  서술의 깊이는 변화의 크기에 비례한다.
```

## 3단계: 축 관련성 — "메가트렌드에 영향을 주는가?"

```
2단계에서 추적한 각 변화에 대해:

질문 (axis.md의 3대 조건을 렌즈로 사용):
  ① 이 변화가 어떤 축의 비가역성을 강화하거나 약화하는가?
     → "되돌리려는 시도가 실패했는가" 또는 "되돌림이 성공했는가"

  ② 이 변화가 어떤 축의 다중 관통 범위를 넓히거나 좁히는가?
     → "새 영역에 진출했는가" 또는 "한 영역에서 이탈했는가"

  ③ 이 변화가 어떤 축의 자기강화 루프를 가속하거나 감속하는가?
     → "루프가 더 빨리 돌고 있는가" 또는 "외부 의존이 드러났는가"

  ④ 이 변화가 레짐(⑨) 환경을 바꾸고 있는가?
     → 금리, 부채, 인플레, 달러 신뢰 — 게임판의 중력 변화.

  ⑤ 이 변화가 배관(구조적 판 ⑧)을 바꾸고 있는가?
     → 공급망, 관세, 기술 디커플링 — 축의 경로가 바뀌는 것.

  ⑥ 어느 축에도 해당하지 않는가?
     → "축 무관"도 유효한 판정. 모든 변화가 축과 연결될 필요 없다.
     → 그러나 "축 무관인데 큰 변화"이면 → 4단계로.

이 단계의 산출물:
  축과 관련된 변화만 추출. 관련성의 내용을 서술.
  태깅은 고정 기호가 아니라 자연어로 — "①AI 자기강화 가속 증거" 등.
```

## 4단계: 이상 탐지 — "내가 모르는 것이 있는가?"

```
이 단계가 닫힌 계를 반개방 계로 만든다.

질문:
  ① 기존 축/Link/Property로 설명 안 되는 변화가 있는가?
     → 있으면 [미분류] 태깅. 버리지 않는다.
     → "설명 안 됨"은 오류가 아니라 발견의 시작.

  ② 평소와 다른 상관관계가 나타나고 있는가?
     → 예: DXY↓ + Gold↓ 동시 = 기존 모델로 설명 불가
     → 예: VIX↑ + HY 안정 = 누가 맞나? → 뭔가 놓치고 있다

  ③ 사용하지 않은 데이터 소스에서 확인해야 할 것이 있는가?
     → 블록체인 축 관련: DeFiLlama TVL? 온체인 NUPL? DEX 볼륨?
     → 재정 레짐 관련: 국채 입찰 데이터? 외국인 보유 변화?
     → 맥락에 따라 추가 MCP 소스를 동원한다.

  ④ 오늘 하나의 오픈 탐색을 한다면 무엇을 검색하겠는가?
     → 고정 쿼리가 아니라, 오늘의 상황에서 가장 불확실한 것을 탐색.

이 단계의 산출물:
  [미분류] 신호 목록 (있으면).
  추가 탐색 필요 영역 (있으면).
  "이상 없음"도 유효한 산출물.
```

## 5단계: 위험 거리 — "가장 가까운 위험은 뭔가?"

```
고정 KC 테이블을 채우는 것이 아니라, 오늘의 데이터에서 위험 거리를 추론한다.

질문:
  ① 현재 가장 가까운 임계점(tipping point)은 무엇인가?
     → axis.md의 KC 목록을 참고하되, 목록에 없는 새 임계점도 식별.

  ② 그 임계점까지의 거리는 좁아지고 있는가, 넓어지고 있는가?
     → 방향(추세)이 절대 거리보다 중요하다.

  ③ 현재 국면(🟢🟡🔴)에서 민감도를 어떻게 조절해야 하는가?
     → 위기 국면에서는 같은 수치라도 의미가 다르다.
     → monitoring.md §12 KC 국면 보정 참조 (배율은 참고선).

이 단계의 산출물:
  "오늘 가장 주시할 임계점" + 거리 + 추세.
  개수는 데이터가 결정한다 (0개도 가능).
```

## 6단계: 전수 점검 — "놓친 것이 없는가?"

```
1~5단계가 자유 추론이라면, 이 단계는 안전망이다.
빈칸을 채우는 것이 아니라, 놓친 것이 없는지 확인만 한다.
문제가 없으면 "확인, 이상 없음"으로 넘어간다.
문제가 있으면 해당 항목을 1~5단계에 추가 반영한다.

[데이터 커버리지]
  □ Macro latest.json을 읽었는가? 신선도는?
  □ P1(통화) 독립 탐색을 했는가?
  □ P2(관세) 독립 탐색을 했는가? (P3에 묻히지 않았는가?)
  □ P3(지정학) 독립 탐색을 했는가?
  □ P4(규제) 독립 탐색을 했는가?
  □ 오늘 맥락에서 추가로 동원해야 할 MCP가 있었는데 안 쓴 것은?

[PSF 3층 커버리지]
  □ S1~S5 중 확인하지 않은 것이 있는가?
  □ F1~F5 중 확인하지 않은 것이 있는가?
  □ Link 상태(L1~L8 + CorrFlip)를 훑었는가? 변화가 있는가?
  □ Divergence(층간 불일치)를 점검했는가?

[축 커버리지]
  □ 1~5단계에서 언급하지 않은 축이 있는가?
     → 언급 안 했다면: 변화가 없어서인가, 놓쳐서인가?
  □ 감속·훼손 신호를 가속 신호보다 먼저 점검했는가?

[이력 연속성]
  □ state.json 직전 관측의 alerts를 확인했는가?
  □ 직전 next_questions에 답했는가 (또는 "미해소" 명시)?
  □ 직전 unclassified 신호가 오늘 해소되었는가, 지속되는가?

발견한 누락이 있으면:
  → 해당 항목을 1~5단계 결과에 보충한다.
  → state.json에 반영한다.
없으면:
  → "전수 점검 완료. 누락 없음." 한 줄로 끝.

[관측/해석 분리 검증] ★ 개선 1
  □ 아래 금지 표현이 보고서에 포함되어 있지 않은가?

  금지 표현 (해석 영역 — CE/TE의 일):
    "~에 베팅하고 있다"
    "~의 의미는"
    "~때문이다" (인과 단정)
    "~할 것이다" (예측)
    "~해야 한다" (행동 권고)
    "~가 전환점이 될 것" (예측)

  허용 표현 (관측 영역 — PSF의 일):
    "[수치], [기간] 변동 [방향]" (사실)
    "[수치], 직전 대비 [변화]" (비교)
    "Link [상태]. 트리거 [충족/미충족]." (기준 대조)
    "[Property A]와 [Property B]가 비동기." (불일치 기술)
    "[현상] — 기존 모델로 설명 불가. [미분류]." (미분류)

  위반 발견 시:
    → 해당 문장을 관측 표현으로 재작성.
    → 해석이 필요하면 "→ CE/TE 해석 요청" 태그 부착.

[사고 오류 방어] R-01~R-08 (errors.md 참조)
  □ R-01: 축 건재 판단 시 PSF 국면도 확인했는가?
  □ R-02: 축 위험 판단 시 3대 조건 중 구체적 약화를 특정했는가?
  □ R-04: 감속·훼손을 가속보다 먼저 점검했는가?
  □ R-07: ⑨재정 지배를 점검했는가?
  (R-03, R-05, R-06, R-08은 W/M Loop에서 점검)

  위반 발견 시: 해당 보고서 부분 재검토 후 수정.
```

## 사용 가능한 데이터 소스 카탈로그

```
맥락에 따라 필요한 소스를 선택한다. 매번 전부 쓸 필요 없다.

[매크로·금리·유동성]
  FRED          : 금리, 스프레드, 실업률, 인플레, 유동성 구성요소
                  → 모든 S층, P-재정, ⑨레짐 관련

[크립토·온체인]
  CoinGecko     : 가격, 시총, 24h 변화
  CoinMetrics   : NUPL, 온체인 메트릭, 투자 스냅샷, 리포트
  blockchain-com: 네트워크 스냅샷, 주소 분석, 매크로 컨텍스트
  DeFiLlama     : TVL, DEX 볼륨, 스테이블코인 상세, 수익률, 프로토콜 분석
  Dune          : RWA 리포트, 체인링크, 커스텀 쿼리
  Etherscan     : 토큰 홀더, 컨트랙트, 링크 분석, 전송 통계

[뉴스·정성 데이터]
  Tavily        : 뉴스 검색, 크롤링, 리서치
                  → P층 독립 수집, 축 관련 이벤트, 오픈 탐색

[한국 시장]
  DART          : 기업 공시, 재무제표, 배당, 검색

소스 선택 원칙:
  → "이 질문에 답하려면 어떤 데이터가 필요한가?"에서 출발.
  → 매번 같은 소스를 쓰는 것이 아니라, 질문이 소스를 결정한다.
  → 새 소스가 추가되면 이 카탈로그에 추가한다.
```

## 산출물 형식

```
형식은 고정하지 않는다. 내용이 형식을 결정한다.

단, 반드시 포함해야 하는 것:
  1. 오늘 발견한 것 — 무엇이 달라졌고, 왜 중요한가
  2. 축과의 관련성 — 있으면 서술, 없으면 "없음"
  3. 설명 안 되는 것 — 있으면 [미분류]로 기록
  4. 가장 가까운 위험 — 임계점과 거리
  5. 다음에 확인할 것 — 질문 형태로

포함하지 않아도 되는 것:
  고정 형식의 테이블, 미리 정해진 이모지 태그,
  변화 없는 축의 반복 서술, 템플릿의 빈칸 채우기.
```
