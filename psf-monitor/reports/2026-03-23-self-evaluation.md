## 주요 문제점

### 1. 관측/해석 경계 위반

```
위반 사례:
  "시장이 짧은 전쟁에 베팅하고 있다" ← 해석
  "S2가 움직이는 순간이 전환점" ← 예측
  "돈은 있는데 쓰지 않는 상태 = 관망" ← 해석

해야 했던 것:
  "HY OAS 327bp, 4주간 변동 0bp" ← 관측만
  왜 그런지는 CE/TE에 위임.
```

### 2. 데이터 신선도 과신

```
FRED 3/18~20 데이터(3~5일 전)로 "전파 확인" 판정.
3/20 이후 변화 가능성 미반영.
"[recent 기반]" 태그 필수화 필요.
```

### 3. "새 발견"의 실체

```
RRP $0.8B: 장기 추세의 일부. "발견"이 아니라 "처음 수집".
Net Liq -$108B: 직전값이 "추정"이므로 차이의 실체 불확실.
```

### 4. 분량 과다

축 모니터링 + PSF 관측 합산 시 의사결정자가 읽기 어려운 분량.
"핵심 3줄"을 맨 앞에 강제하는 구조 필요.

### 5. 축/PSF 이중 실행 비효율

같은 데이터를 두 번 서술. 하나로 합쳐야 함.

## 개선 방향

1. **관측/해석 분리 강제**: "왜", "~에 베팅", "~의 의미는" 표현 금지. 관측만.
2. **분량 제한**: 핵심 3줄이 보고서를 대표. 상세는 state.json에만.
3. **축+PSF 통합**: 한 번의 실행에서 두 렌즈 모두 적용. 중복 제거.
4. **신선도 정직**: "[recent 기반]" 태그 필수. 추정값 명시적 구분.
