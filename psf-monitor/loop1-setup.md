## 운영 규칙

### 기록 시점
모든 psf-monitor 보고서 및 macro-weekly 보고서 발행 시:
  1. 핵심 결론에서 검증 가능한 예측 문장을 추출
  2. PM_핵심예측에 기록
  3. PM_검증일 설정

### 검증 시점
매주 일요일 (주간 리뷰 시):
  1. PM_검증일 ≤ 오늘인 항목 필터
  2. 실제 결과 수집 (MCP 사용)
  3. 적중/부분적중/오판 판정

### 적중 기준
  방향(↑↓) 일치 → 1차 기준
  크기(±20% 이내) → 2차 기준
  시기(±2주) → 3차 기준 (Type D 판별)

## 기존 Property와의 관계

| 기존 Property | LOOP 1 역할 | 비고 |
|---|---|---|
| `핵심 결론` | PM_핵심예측의 원천 | 결론에서 예측 문장 추출 |
| `결과확인 예정일` | PM_검증일과 유사 | PM_검증일은 post-mortem 전용 |
| `검증 판정` | 기존 검증(데이터 정합성) | PM_검증상태는 예측 적중 여부 |
| `판정 정확` | 단순 Y/N | PM_검증상태가 4단계로 세분화 |
| `실패 원인` | 데이터 레벨 실패 | PM_오류유형은 판단 레벨 분류 |

→ 기존 Property는 **분석 품질 검증** (데이터가 맞는가)
→ PM_ Property는 **예측 검증** (판단이 맞았는가)
→ 두 축은 독립. 병행 운영.
