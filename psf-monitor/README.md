# PSF Monitor — 판·구조·흐름 관측 모듈

> 3층 계기판 관측 + 메가트렌드 축 모니터링 통합.
> 개방형 추론(1~5단계) + 전수 점검(6단계) 하이브리드.

---

## 파일 구조

```
psf-monitor/
├── CLAUDE.md              ← 정체성 + 불변 원칙
├── SKILL.md               ← 실행 프로토콜 (적응형)
├── axis-map.md            ← ★ 신호 발견 프로토콜 (6단계)
├── monitoring.md          ← 관찰 리듬 (일일/주간/월간/분기)
├── axis.md                ← 축 정의서
├── 원본-PSF 온톨로지 엔진.md  ← PSF 온톨로지 v3
├── ontology.md            ← 운용 헌법
├── collection.md          ← 데이터 매핑
├── errors.md              ← 셀프검증 체크리스트
├── state.json             ← 현재 관측 (개방형 스키마)
├── history/               ← 관측 이력
│   └── 2026-03-23.json
├── reports/               ← 보고서 + 평가
│   ├── 2026-03-23-psf-observation.md
│   └── 2026-03-23-self-evaluation.md
└── README.md
```

---

## 실행 흐름

```
데이터 수집 → 1~5단계(개방형 추론) → 6단계(전수 점검)
→ state.json 저장 → history 스냅샷 → reports 보고서
```

## 렌즈

```
하나의 프로토콜, 두 개의 렌즈:
  PSF 렌즈: "충격이 3층을 어떻게 통과하는가"
  축 렌즈: "메가트렌드 방향이 바뀌고 있는가"
```

## 알려진 개선 과제

1. 관측/해석 분리 강제 장치 필요
2. 보고서 분량 제한 (핵심 3줄 선행)
3. 축+PSF 통합 실행 (이중 실행 제거)
4. 데이터 신선도 정직 태깅
