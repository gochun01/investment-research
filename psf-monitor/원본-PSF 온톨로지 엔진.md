# 판-구조-흐름 온톨로지 엔진 v3 — 프로젝트 지침

## 이 지침의 목적

이벤트가 들어오면 아래 프레임워크로 즉시 분석한다.
매번 온톨로지를 설명하지 않는다. 이미 알고 있는 것으로 작동한다.

> **v3 변경**: P층 Property를 하위 속성(P1-a/b/c 등) + 4단계 판정 + 합산 로직으로 확장.
> 런타임 데이터(현재 상태, 이벤트 로그)를 state.json/Notion으로 이관.
> 축(axis.md)과의 관계 명시.

---

## 핵심 비유: 시장은 3층 건물

- 3층(판): 룰이 바뀌었나? → 정책 레짐, 룰 충격
- 2층(구조): 건물이 버티나? → 금리 구조, 신용 구조
- 1층(흐름): 돈이 어디로 가나? → 유동성 방향, 포지션 상태
- 정상: 물은 위→아래 흐름 (순방향)
- 위기: 아래→위 역류 (역방향 L7, L8)

---

## 궁극적 질문 3개

1. 지금 돈을 넣어도 되는 세상인가? (L7/L8 확인)
2. 어디에 넣는 게 유리한 세상인가? (바뀐 객체→경로 추적)
3. 언제 바뀌는가? (트리거 시차 확인)

## 7개 구성요소

### ① Object (6개) — 무엇을

| ID | 이름 | 층 | 역할 |
|----|------|-----|------|
| OBJ-01 | 정책 레짐 | 판 | 중앙은행이 돈의 가격을 정한다 |
| OBJ-02 | 룰 충격 | 판 | 관세·지정학·규제가 룰을 바꾼다 |
| OBJ-03 | 금리 구조 | 구조 | 돈의 가격이 시장에 전달된다 |
| OBJ-04 | 신용 구조 | 구조 | 시스템이 충격을 견디는가 |
| OBJ-05 | 유동성 방향 | 흐름 | 돈이 어디로 가는가 |
| OBJ-06 | 포지션 상태 | 흐름 | 돈이 실제로 갔는가 |

### ② Property (14개) — 어떤 상태인지

**판(P) — 게임의 룰. 축(axis.md)의 힘이 통과하는 조건.** [v3 확장]

> 판의 3가지 본질적 속성:
> ① 비대칭성 — 시장 참여자가 통제할 수 없는 외생변수
> ② 불연속성 — 점진적이 아니라 계단식 변화. KC가 판 층에 집중되는 이유
> ③ 선행성 — P→S→F 인과 방향이 기본 (역방향 L7/L8도 존재)

**★ 독립 수집 원칙 [v3 신규]:**
```
각 Property(P1~P4)는 독립적으로 수집하고, 독립적으로 판정한다.
├── 하나의 지배 이벤트(전쟁, 금융위기)가 다른 속성의 판독을 덮지 못하게 한다
├── P3(전쟁)이 🔴여도 P2(관세)를 별도로 수집·판정한다
├── "글로벌 매크로 뉴스" 같은 통합 검색으로 퉁치지 않는다
└── 각 Property마다 전용 검색 쿼리를 실행한다 (SKILL.md §P-Layer 쿼리 참조)

이유: 지배 이벤트가 헤드라인을 지배하면 다른 속성 뉴스가 밀린다.
     전쟁 중에도 관세율은 바뀌고 규제는 진행된다.
     이 "묻힌 변화"를 놓치면 뒤늦게 판 전체가 바뀌어 있음을 발견한다. (F-01 교훈)
```

**P1 통화정책** (OBJ-01 정책 레짐):

```
P1-a  Fed 금리 경로
      │ 지표: DFF, 점도표 중앙값, CME FedWatch 내재 확률
      │ 의미: 자금의 기본 가격. 모든 자산 가치의 할인율 기초.
      └ 감시: FOMC (연 8회) + 의사록 + 위원 발언

P1-b  QT/QE 상태 및 방향
      │ 지표: Fed 대차대조표 규모, 월간 축소/확대 속도, RRP 잔고
      │ 의미: 시스템 내 유동성 총량. 금리와 별도로 자산 가격에 직접 영향.
      └ 감시: 주간 Fed 대차대조표, RRP 일별 잔고

P1-c  글로벌 중앙은행 동조/이탈
      │ 지표: ECB/BOJ/PBOC/BOE 금리 결정, 통화정책 방향
      │ 의미: 동조 = 글로벌 유동성 동시 확대/축소. 이탈 = 환율+캐리 영향.
      └ 감시: 각국 CB 회의, 특히 BOJ(엔캐리), PBOC(위안 관리)
```

| 판정 | 조건 |
|------|------|
| 🟢 정상 | Fed 경로가 시장 기대와 일치. QT/QE 방향 예측 가능. 글로벌 동조. |
| 🟡 경계 | 점도표-시장 괴리 >50bp. QT 속도 변경 시사. 1개+ 주요 CB 이탈. |
| 🔴 위기 | 긴급 금리 변경(인터미팅). QE 재개/QT 중단. 글로벌 CB 정책 분기. |
| ⚫ 없음 | FOMC 간 공백기 + 위원 발언 없음. 데이터 미확보. |

```
P2-a  관세율 변동 및 예고

P2-b  무역협정/제재 변화

P2-c  공급망 재편 방향
```

| 판정 | 조건 |
|------|------|
| 🟢 정상 | 관세율 안정. 신규 제재 없음. 공급망 정책 예측 가능. |
| 🟡 경계 | 관세율 5%p+ 변동 예고. 신규 제재 발표. 보복 관세 우려. |
| 🔴 위기 | 대규모 관세 즉시 시행(10%p+). 주요 교역국 무역 전면 중단 위협. Entity List 대폭 확대. |
| ⚫ 없음 | 관세 관련 뉴스 부재 (검색 확인 완료). |

> ⚠ P2는 P3(전쟁)에 묻히기 가장 쉬운 속성. P2 전용 쿼리 별도 실행 필수.
> "특이사항 없음"도 반드시 출력 명시. 검색 없이 생략 금지.

**P3 지정학 리스크** (OBJ-02 룰 충격):

```
P3-a  군사적 충돌/긴장 수준
      │ 지표: 활성 분쟁 목록, 군사 배치 변화, 교전 강도
      │ 의미: 직접적 공급 차질 + 리스크 프리미엄 + 안전자산 수요

P3-c  제재·외교 국면 변화
```

| 판정 | 조건 |
|------|------|
| 🟢 정상 | 활성 분쟁 없음 or 동결. 공급 경로 정상. 외교 안정. |
| 🟡 경계 | 기존 분쟁 확대 우려. 공급 경로 부분 위협. 제재 강화 예고. |
| 🔴 위기 | 신규 군사 충돌. 주요 해협 폐쇄/통행 제한. 대규모 제재 즉시 발효. |
| ⚫ 없음 | 지정학 뉴스 부재 (매우 드묾). |

> P3 🔴 + L3.5 비활성 = 사건은 발생했으나 시장 전파 미도달. 이 간극이 선제 대비의 창.

### ③ Link (8개) — 어떻게 연결되는지

**순방향 (위→아래, 정상):**
- L1 정책→금리: 돈의 가격이 바뀌었다
- L2 룰 충격→신용: 시스템이 충격을 견디는가
- L3 룰 충격→금리: 충격이 물가로 번지는가 (가장 느리고 불확실)
- L4 금리→유동성: 돈이 어디로 가는가
- L5 신용→유동성: 돈이 갈 수 있는가 (게이트 역할)
- L6 유동성→포지션: 돈이 실제로 갔는가

**역방향 (아래→위, 위기):**
- L7 포지션→신용: 한 사람의 문제가 모두의 문제가 되는가
- L8 신용→정책: 시장이 정부를 굴복시키는가

### ④ Trigger — 언제 작동하는지

트리거 = IF (조건 조합) → THEN (실행)
**개별 트리거가 주(主) 판정 기준. 복합 지수는 보조.**

**L1 정책→금리:**
- IF: DFF 변경 + 점도표 이동
- 시차: 즉시~1주
- 확인: T10Y2Y 방향, DFII10 변동

**L2 룰 충격→신용:**
- IF: 유효관세율 ±5%p + 대상국 주요 변경
- 시차: 1~4주
- 확인: HYG 주간 ≤-2%, HYG/IEF 비율 하락 >2%

**L3 룰 충격→금리:**
- IF: 유효관세율 ±5%p + BEI(T10YIE) ±0.3%p
- 시차: 2~8주 (가장 느림)
- 확인: BEI 방향, CPI 에너지

**L4 금리→유동성:**
- IF: 실질금리(DFII10) 1.0% 교차 + 10Y2Y 방향 전환
- 시차: 2~8주
- 확인: DXY ±3%(4주), MMF 잔고 ±2%

**L5 신용→유동성 (게이트):**
- IF: HYG/IEF 4주 최저 하회 + SOFR-FF >30bp
- 시차: 즉시~1주
- 확인: 회사채 발행 정지 여부 (수동)

**L6 유동성→포지션:**
- IF: DXY ±3%(4주) + EEM/SPY 2주 연속 방향 전환
- 시차: 1~4주
- 확인: SPY 거래량 20일 평균 150%, 스테이블 시총 ±3%

**L7 포지션→신용 (역방향 ⚠):**
- IF: VIX >30 + 3일 유지 + HYG 1일 -1.5%
- 시차: 즉시
- 확인: SOFR-FF >20bp 동시
- **→ 국면 🟡 경계**

**L8 신용→정책 (역방향 🚨):**
- IF: LQD 주간 -5% + 2주 발행 정지(수동) + SOFR-FF >50bp
- 시차: 즉시
- 확인: CP 시장 동결, Fed 긴급 시설 발동
- **→ 국면 🔴 위기**

### ⑤ Function (복합 판정 지수) — 보조 지표 [v2 신규]

**원칙: 개별 트리거가 주(主), 복합 지수는 보조(補).**
복합 지수는 개별 트리거의 사각지대를 잡는 "감각 보정용"이다.
의사결정의 근거가 아니라 "뭔가 놓치고 있나?" 확인용이다.
Action Log가 충분히 쌓이면(6개월+) 가중치를 실증 교정하여 판정용으로 승격 검토.

**L7 경계 점수 (참고용):**
```
L7_score = VIX_norm × 0.4 + HYG_weekly_norm × 0.3 + SOFR_FF_norm × 0.3

정규화: 각 지표를 0~1로 변환
  VIX_norm = min(VIX / 40, 1)
  HYG_weekly_norm = min(abs(HYG_weekly_chg) / 5%, 1)  (하락만)
  SOFR_FF_norm = min(SOFR_FF_gap / 50bp, 1)

해석:
  0.0~0.3 → 정상 (참고 불필요)
  0.3~0.6 → 주의 (개별 트리거 중 하나가 접근 중일 수 있음)
  0.6~0.8 → 감시 강화 (개별 트리거 미충족이라도 주의)
  0.8~1.0 → 트리거 임박 (개별 트리거와 교차 확인)
```

**L8 위기 점수 (참고용):**
```
L8_score = LQD_weekly_norm × 0.4 + SOFR_FF_norm × 0.4 + issuance_norm × 0.2

정규화:
  LQD_weekly_norm = min(abs(LQD_weekly_chg) / 8%, 1)  (하락만)
  SOFR_FF_norm = min(SOFR_FF_gap / 80bp, 1)
  issuance_norm = 0 (발행 정상) / 0.5 (발행 감소) / 1.0 (발행 정지)

해석: L7과 동일 구간
```

**사용법:**
- 복합 점수가 0.6 이상인데 개별 트리거가 하나도 안 걸렸으면 → "뭔가 놓치고 있나?" 점검
- 개별 트리거가 충족됐는데 복합 점수가 0.3 이하면 → "잡음인가?" 재확인
- 복합 점수끼리 비교: "이번 이벤트(0.45)는 2024.08 엔캐리(0.78)보다 아직 낮다"

**가중치 현황:**
```
현재: 초기값 (과거 위기 4건 관찰 기반, 통계적 검증 없음)
교정 예정: Action Log 50건+ 축적 시 (예상 6개월 후)
교정 방법: 트리거 판정이 맞았던/틀렸던 사례에서 각 변수의 실제 기여도 역산
```

### ⑥ Interface (층별 통합 뷰) [v2 신규]

3개 층을 한 문장으로 읽는 렌즈.
이벤트 분석 시 "판 전체 / 구조 전체 / 흐름 전체"를 먼저 읽고, 필요시 개별 객체로 내려간다.

```
판 (OBJ-01 + OBJ-02):
  공통 질문: "게임의 룰이 바뀌고 있는가?"
  상태 요약: 안정 / 변동 / 전환

구조 (OBJ-03 + OBJ-04):
  공통 질문: "건물이 버티고 있는가?"
  상태 요약: 건전 / 긴장 / 균열

흐름 (OBJ-05 + OBJ-06):
  공통 질문: "돈이 움직이고 있는가?"
  상태 요약: 정체 / 이동 / 이탈
```

### ⑦ Action (3개 + 확장) — 그래서 뭘 하는지

**기본 3개:**
1. **충격 전파 추적**: 이벤트 → 경로 → 어디까지 가는가
2. **위기 경보**: L7/L8 활성 여부 → 국면 판정
3. **주간 상태 판독**: 지금 내 위치는 어디인가

**확장 Action [v2 신규]:**
4. **시나리오 시뮬레이션**: 이벤트별 분기 시나리오와 속성값 변동 추정
5. **판정 기록**: 모든 분석을 Action Log에 기록, 후일 정확도 검증

## 국면 판정 — 3초 룰

```
🟢 정상: L7 ❌ L8 ❌ → 위험자산 유지, 섹터 선택
🟡 경계: L7 ✓ L8 ❌ → 위험자산 축소, L8 집중 모니터
🔴 위기: L7 ✓ L8 ✓ → 현금 극대화, 정책 대응 후 재진입

보조 확인 (참고용):
  L7_score > 0.6 + 개별 트리거 미충족 → "놓치고 있는 것 없나?" 점검
  L7_score < 0.3 + 개별 트리거 충족 → "잡음인가?" 재확인
```

## 이벤트 분석 파이프라인 [v2 확장]

이벤트가 들어오면 이 순서로 분석한다:

```
1단계: 이벤트 → 어느 Object에 꽂히나? 어떤 Property가 바뀌나?
2단계: Interface로 층 전체 상태 확인 → 개별 Link 확인
3단계: 트리거 조건 충족하나? (IF 조건 조합 확인)
       → 확실하면 바로 판정
       → 애매하면 데이터로 확인 (MCP 서버: FRED, Yahoo, CoinGecko)
       → 복합 점수 참고 확인 (사각지대 체크)
4단계: 판정
       → 건물 흔드나? (구조층 도달 여부)
       → 가구 배치만 바뀌나? (1층 섹터 회전)
5단계: 시나리오 분기표 작성
       → 2~3개 시나리오별 속성값 변동 추정
       → 각 시나리오별 트리거 충족 여부 사전 판정
       → 시나리오별 선점 포지션 명시
6단계: Action + 기록
       → 매크로 변경 / 섹터 판단 / 선점 포지션 제시
       → Action Log에 기록 (판정 근거, 확인 시점 포함)
```

## 분석 출력 형식 [v2 확장]

이벤트 분석 시 아래 형식으로 출력한다:

```
## 이벤트: [이벤트 내용]

### 1. Object-Property
[어느 객체, 어떤 속성이 변했는지]
층 상태: 판 [안정/변동/전환] | 구조 [건전/긴장/균열] | 흐름 [정체/이동/이탈]

### 2. Link-Trigger
[어떤 Link가 열리는지, 트리거 충족 여부]
- L_ : IF [조건] → [충족/미충족]
- 확인 지표: [데이터]
- 복합 점수 (참고): L7_score = X.XX, L8_score = X.XX

### 3. 판정
[건물 흔드나 / 섹터 회전인가 / 변화 없나]
국면: 🟢/🟡/🔴

### 4. 시나리오 분기표
              │ 현재  │ A: [시나리오A] │ B: [시나리오B] │ C: [시나리오C]
──────────────┼──────┼──────────────┼──────────────┼──────────────
[핵심 속성1]   │ 값   │ 추정값        │ 추정값        │ 추정값
[핵심 속성2]   │ 값   │ 추정값        │ 추정값        │ 추정값
L_ 트리거     │ 상태 │ 충족/미충족    │ 충족/미충족    │ 충족/미충족
국면          │ 🟢   │ ?             │ ?             │ ?
선점          │ -    │ [행동]        │ [행동]        │ [행동]

### 5. Action
[매크로 변경 여부 + 섹터 판단 + 선점 포지션]

### 6. 다음 확인
[확인 지표 + 시점 + 시나리오 분기 트리거]

### 7. Action Log 기록
날짜 | 이벤트 | 판정 | 근거 | 확인시점 | (후일 기입: 실제결과, 정확도, 교훈)
```

## 정성 우선 원칙

1. 정성적으로 먼저 판단한다 ("건물 흔드나?" → 5초)
2. 데이터는 그 판단을 확인·부정하는 근거다
3. 확실하면 데이터 확인 불필요, 애매할 때만 데이터가 판정관
4. MCP 서버(FRED, Yahoo Finance, CoinGecko, CoinMetrics)로 즉시 검증 가능
5. 복합 지수는 참고용. 개별 트리거와 불일치 시에만 주목한다 [v2]
6. 시나리오 분기표의 숫자는 범위 추정이지 정밀 예측이 아니다 [v2]

| 컬럼 | 설명 | 기입 시점 |

| 객체 | 어느 Object에 꽂혔나 | 즉시 |
| Link | 확인한 Link | 즉시 |

| 복합점수 | L7/L8 score (참고) | 즉시 |
| 확인시점 | 언제 결과를 검증하나 | 즉시 |

## Automation 계층 [v2 신규]

대시보드 구축 시 적용. 3단계 자동화.

```
Level 1 (자동 알림 — 사람 개입 없음):
  속성값이 임계값의 70%에 도달 시 자동 알림
  "VIX 25 돌파. L7 감시 구간 접근."
  "HYG 주간 -1.5%. L7 트리거 -1.5% 기준의 100%."

Level 2 (반자동 제안 — 사람 확인 필요):
  트리거 조건 조합 충족 시 사전 정의된 액션 제안
  "L7 트리거 충족. 위험자산 비중 20% 축소를 제안합니다. 확인하시겠습니까?"

Level 3 (수동 실행 — 사람이 최종 결정):
  최종 실행은 항상 사용자 확인 후
  "확인 → 실행" 또는 "거부 → 사유 기록"
```
