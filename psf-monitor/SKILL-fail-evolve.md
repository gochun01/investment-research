## 실패 패턴과 방어 (Step 6: FAIL)

> 정상 경로만으로는 80점. 방어 경로가 있어야 90점+.
> 상세 오류 이력: errors.md 참조. 여기는 F-테이블(구조적 방어).

### 오류 유형 패턴 (12건 분석)

```

  누락 → ontology ↔ collection 1:1 교차 검증 + validate.py 자동화
```

## 진화 규칙 (Step 7: EVOLVE)

> 지침이 진화하면 LLM이 학습한 것과 같다.

### Self-Audit (매 관측 완료 후)

```
Q1. 전수 점검: P1~P5, S1~S5, F1~F5 전부 state.json에 있는가?
    → 하나라도 빠지면 ⚠. 빠진 이유를 "[미확인 — 사유]"로 기록.

Q2. 출처 정합성: 모든 observation에 [MCP:소스] 또는 [뉴스:출처] 태그가 있는가?
    → quality.mcp_ratio < 50%이면 ⚠.

Q3. 관측/해석 분리: 보고서에서 은유·의인화·가치판단 문장이 있는가?
    → "위험하다", "좋다", "~에 베팅", "~을 먹이고" 등 발견 시 ⚠.
    → 재서술: "VIX 26.78, 경계 구간" (O) / "위험하다" (X)

Q4. 감속 우선: 가속 신호를 먼저 서술하고 감속 신호를 나중에 서술했는가?
    → 감속을 먼저 점검했는가? (불변 원칙 7, F-11 방지)

Q5. 미분류 보존: 설명 안 되는 신호를 버리지 않고 [미분류]로 기록했는가?
    → unclassified에 0건인데 "없었다"로 넘어갔으면 ⚠.
```

### 이슈 적재 조건

```
일회성: 이번 관측만의 문제 → 보고서 내 수정 후 끝.
시스템성: 동일 ⚠ 2회+ 발생 → errors.md에 ERR-xxx 추가 + F-테이블 갱신.
```

### 갱신 주기

```
일일 (D Loop): Self-Audit Q1~Q5 실행.
주간 (W Loop): errors.md 신규 오류 리뷰. F-테이블 패턴 확인.
분기 (Q Loop): ontology/axis 개정 가능. F-테이블 전수 리뷰.
              감사 결과 → audits/YYYY-QN.md 저장.
```

### 갱신 주체

```
자동 (LLM): Self-Audit 실행, 오류 기록, F-테이블 발생 횟수 갱신.
승인 필요: F-테이블 방어 규칙 추가/수정, SKILL 실행 순서 변경.
사용자: ontology/axis 개정, 불변 규칙 변경, GUARDRAILS.md 수정.
```

### 불변 규칙 (진화의 하한선)

```
1. PSF 3층(판-구조-흐름) 구조 자체를 해체하지 않는다.
2. S1~S5, F1~F5 전수 수집을 "주요 N개만"으로 축소하지 않는다.
3. 관측과 해석의 분리를 훼손하지 않는다.
4. F-테이블을 삭제하지 않는다. 누적만 한다.
5. Self-Audit에서 "불편한 질문"(Q3, Q4)을 제거하지 않는다.
6. errors.md의 오류 기록을 삭제하지 않는다.
7. 원본-PSF 온톨로지 엔진.md의 최고 우선순위를 변경하지 않는다.
8. "데이터가 과정과 답을 결정한다"는 열린 구조 원칙을 폐기하지 않는다.
```

## 7단계 프로토콜 채점 (psf-monitor)

```
Step 1 WHY:    ✅ "세계의 현재 상태를 관측하고 축까지의 거리를 측정"
Step 2 WHAT:   ✅ IN(관측)/OUT(해석·예측·행동권고)/ADJACENT(CE·TE·macro)
Step 3 WHEN:   ✅ D/W/M/Q 트리거 + 긴급 + 수시 + 금지(해석·예측)
Step 4 HOW:    ✅ Phase 0 → 수집 → axis-map 6단계 → 기록 → 출력
Step 5 FORMAT: ✅ 텍스트 20줄 + HTML + state.json. 좋은/나쁜 예시(CLAUDE.md)
Step 6 FAIL:   ✅ F-01~F-12 테이블 + 오류 유형 패턴 분석
Step 7 EVOLVE: ✅ Self-Audit Q1~Q5 + 갱신 주기/주체 + 불변 규칙 8개

점수: 7/7 → 정밀도 90점+ (자기 진화 가능)
```
