# 투영 엔진 — 12개월 시나리오 프로토콜

> **목적**: 3시스템(macro+PSF+축)의 관측을 입력으로, 12개월 시나리오를 산출한다.
> **원칙**: 관측 시스템과 동일하게 개방형. 시나리오는 데이터가 결정한다.
> **주기**: 분기 1회 전면 구성 (Q Loop). 월간 확률 조정. 일일은 분기 조건 충족 시에만.

---

## 원칙

```
1. 예측이 아니라 시나리오다
   "12개월 후 이렇게 된다"가 아니라
   "이런 조건이면 이 경로, 저런 조건이면 저 경로" 분기.
   확률은 확신이 아니라 현재 정보의 가중치.

2. 관측이 투영을 먹인다
   macro state + PSF state + axis state → 투영 입력.
   관측이 stale이면 투영도 stale.

3. 시나리오는 살아 있다
   이벤트가 분기 조건을 충족하면 즉시 확률 재조정.
   분기마다 시나리오 자체를 재구성.

4. 틀리는 것이 정상이다
   12개월 예측은 틀린다. 핵심은 "어떻게 틀렸는가"에서 배우는 것.
   PM_핵심예측으로 검증. 틀린 패턴이 다음 시나리오를 개선.
```

---

## 구성 요소 1: 시나리오 프레임워크

```
질문:
  ① 현재 조건에서 12개월 내 가능한 경로는 몇 개인가?
     → 보통 3~4개. 너무 많으면 구분이 무의미.

  ② 각 경로를 가르는 분기 조건(trigger)은 무엇인가?
     → 관측 가능하고 판정 가능한 조건이어야 한다.
     → "시장 심리 변화" (X) → "VIX 30 돌파 + HY 400bp+" (O)

  ③ 각 경로의 확률은?
     → 100% 합산. 가장 높은 것이 Base.
     → 확률 근거를 1줄로 명시.

  ④ 각 경로의 시간 경과는?
     → 3개월 / 6개월 / 12개월 후 상태.
     → macro 레짐(🟢🟡🔴), PSF 국면, 핵심 지표 범위.

산출물:
  시나리오 테이블 (경로 × 시점 매트릭스)
```

---

## 구성 요소 2: 사이클 포지셔너

```
질문:
  ① 경기 사이클: 확장 초기 / 중기 / 후기 / 둔화 / 수축 / 회복 중 어디?
     → 입력: GDP, PMI, 실업률, Core PCE 조합.

  ② 신용 사이클: 완화 / 중립 / 긴축 중 어디?
     → 입력: HY OAS, SLOOS, 은행 지급준비금.

  ③ 유동성 사이클: 확장 / 정점 / 수축 / 바닥 중 어디?
     → 입력: Net Liq, RRP, TGA, Fed 대차대조표 방향.

  ④ 각 사이클에서 "여기서 12개월 후"의 일반적 경로는?
     → 과거 패턴 참조. 단, 현재 고유 조건(전쟁, 교란)을 반영.

산출물:
  3개 사이클의 현재 위치 + 12개월 투영 방향
```

---

## 구성 요소 3: 축 속도 투영

```
질문:
  ① 각 축의 S커브 현재 위치에서, 12개월 후 이동 가능 범위는?
     → 촉매가 있으면 단계 이동. 없으면 현 단계 유지.

  ② 12개월 내 예정된 촉매는 무엇인가?
     → 규제 일정, 기술 발표, 정책 시한 등.
     → 각 촉매의 실현 확률.

  ③ 축 간 교차가 12개월 내 새로 열리거나 닫히는가?
     → ①AI × ②에너지 (전력 병목) → 악화? 완화?
     → ⑨재정 × ④블록체인 (달러 대안) → 가속?

산출물:
  축별 12개월 투영 (S커브 이동 + 촉매 + 확률)
```

---

## 구성 요소 4: 피드백 루프 시뮬레이션

```
질문:
  ① 현재 활성 경로(교란, 정상)가 N개월 지속되면 어떤 상태에 도달하는가?
     → 시간 투영: 1개월 / 3개월 / 6개월 / 12개월.

  ② 경로가 자기강화인가, 자기소멸인가?
     → 자기강화: 지속될수록 악화 (예: 유가→인플레→금리차→엔약세→유가)
     → 자기소멸: 지속될수록 반전 압력 (예: 유가↑→수요파괴→유가↓)

  ③ 경로를 끊는 외생 개입은 무엇인가?
     → 정책(Fed 긴급 인하), 외교(전쟁 종결), 시장(캐리 청산).

산출물:
  활성 경로별 시간 투영 + 자기강화/소멸 판정 + 개입 트리거
```

---

## 구성 요소 5: 역사 패턴 참조

```
질문:
  ① 현재와 가장 유사한 과거 국면은?
     → 매칭 기준: 레짐, 교란 유형, 사이클 위치, 지정학.

  ② 그 국면에서 12개월 후 어떻게 됐는가?
     → 레짐 경로, 자산 수익률, 축 진행.

  ③ 현재와 과거의 핵심 차이는?
     → 차이가 경로를 바꿀 수 있는 부분.

산출물:
  유사 국면 1~3개 + 12개월 결과 + 현재와의 차이
```

---

## 구성 요소 6: 자산 함의 매핑

```
질문:
  ① 각 시나리오에서 자산 클래스별 방향은?
     → 위험자산(주식), 안전자산(금/국채), 대안(BTC/크립토), 현금.

  ② 확률 가중 기대값은?
     → 시나리오 확률 × 자산 방향 = 기대값.

  ③ 축(메가트렌드)과 연결된 구조적 기회는?
     → 시나리오와 무관하게 축이 밀어주는 영역.
     → 시나리오에 따라 타이밍만 다른 것.

산출물:
  시나리오별 자산 테이블 + 확률 가중 + 구조적 기회
```

---

## 실행 주기

```
분기 (Q Loop): 전면 구성. 6개 구성 요소 모두 실행. 12개월 시계 리셋.
월간 (M Loop): 확률 재조정. 분기 조건 충족 여부 점검.
일일 (D Loop): 분기 조건 충족 시에만 "투영 변화" 1줄 추가.
긴급: 시나리오 분기 이벤트 발생 시 즉시 재조정.

산출물 저장:
  projection.json — 현재 시나리오 + 확률 + 사이클 + 축 속도
  reports/YYYY-QN-projection.html — 분기 투영 보고서 (HTML)
```

## 투영 보고서 형식

```
분기 투영은 기존 일일 보고서와 별도.

구조:
  [현재 위치] 3시스템 상태 요약 1줄
  [시나리오 테이블] 경로 × 시점 매트릭스
  [시나리오별 해석] 각 경로 2~3문장 (개인투자자 수준)
  [사이클] 경기/신용/유동성 현재 위치 + 12개월 방향
  [축 속도] 축별 S커브 12개월 투영
  [자산 함의] 확률 가중 기대값
  [검증 일정] PM 예측 + 분기 조건 목록
```

> **입력**: macro/indicators/latest.json + psf-monitor/state.json + axis.md
> **출력**: projection.json + 분기 투영 보고서 (HTML)
