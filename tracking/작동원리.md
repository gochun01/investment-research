# Stereo Analyzer 작동 원리

> 기사·쟁점·이슈를 먼저 판독하고 → 7-Layer로 해부하고 → 되먹임으로 보강하고 → 불확실성을 고백한다
>
> 이 문서는 `LLM기반 시스템 표준 아키텍처`를 Stereo Analyzer에 적용한 것이다.

---

## 2. 5-Layer 스택

```
┌──────────────────────────────────────────┐
│ Layer 5: 출력 (Output)                    │
│  텍스트 템플릿 (SKILL.md 형식)            │
│  HTML 보고서 (render_adaptive.py)         │
│  Notion 저장                             │
├──────────────────────────────────────────┤
│ Layer 4: 판단 (Judgment)                  │
│  Pre-Read 라우팅 결정                     │
│  Layer별 가중치 자율 배분                  │
│  시그널/노이즈 판별                       │
│  FB-4 최종 대조                           │
├──────────────────────────────────────────┤
│ Layer 3: 분석 (Analysis)                  │
│  7-Layer 해부 (L1~L7)                    │
│  되먹임 순회 (FB-1~FB-4)                  │
│  돌발 질문 생성                           │
│  불확실성 지도 작성                       │
├──────────────────────────────────────────┤
│ Layer 2: 수집 (Collection)                │
│  Firecrawl → 기사 전문                   │
│  Tavily → 관련 기사 2~3건                │
│  Notion → 이전 분석 검색                  │
├──────────────────────────────────────────┤
│ Layer 1: 입력 (Input)                     │
│  URL / 텍스트 / 키워드 / 질문             │
│  감정 감지 → 감정 분리 트리거              │
└──────────────────────────────────────────┘
```

## 3. 세션 생명주기 (Session Lifecycle)

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│  Phase 0: Pre-Read & Context Routing                            │
│  ├── 0-1. Type 분류 (POLICY/MACRO/STRUCT/EVENT/NARR/NOISE)     │
│  ├── 0-2. SCP 판독 (0~5)                                       │
│  ├── 0-3. Urgency 판정 (URGENT/WATCH/SLOW)                     │
│  ├── 0-4. 감정 분리 (해당 시)                                    │
│  └── 0-5. 분석 전략 결정 (어디에 집중/축소/생략)                   │
│           ↓                                                     │
│  Phase 1: 수집                                                  │
│  ├── Firecrawl → 기사 전문                                      │
│  ├── Tavily → 관련 기사 2~3건                                   │
│  └── Notion → 이전 분석 검색                                    │
│           ↓                                                     │
│  Phase 2: 7-Layer 해부 (가중치 적용)                              │
│  ├── SURFACE: L1(헤드라인) → L2(팩트) → L3(이해관계)             │
│  └── SUBSURFACE: L4(인과) → L5(구조) → L6(2차효과) → L7(판단)   │
│           ↓                                                     │
│  Phase 2.5: 되먹임 순회 (Feedback Loop)                          │
│  ├── FB-1: L4→L2 (정책/매크로/내러티브일 때)                     │
│  ├── FB-2: L5→L3 (구조/정책일 때)                                │
│  ├── FB-3: L6→L7 (SCP≥3일 때)                                   │
│  └── FB-4: L7→L1 (항상 실행. "처음과 다르게 보이는가?")           │
│           ↓                                                     │
│  Phase 3: 돌발 질문 생성 (Emergent Questions)                    │
│  ├── 5가지 렌즈 중 적합한 것 선택                                │
│  └── 최소 1개, 최대 3개 질문 + 답변 시도                          │
│           ↓                                                     │
│  Phase 4: 불확실성 지도 (Uncertainty Map)                        │
│  ├── Layer별 확신도 자가 진단 (1~5)                               │
│  ├── 가장 약한 고리 식별                                         │
│  └── 보강 방법 제시                                              │
│           ↓                                                     │
│  출력:                                                          │
│  ├── 텍스트 출력 (SKILL.md 출력 템플릿)                           │
│  ├── HTML 보고서 (render_adaptive.py)                            │
│  ├── history/ 스냅샷 저장                                        │
│  └── 셀프검증 체크리스트 실행                                    │
│                                                                 │
│  NOISE 분기:                                                    │
│  Phase 0에서 NOISE 판정 → L1만 출력 → 종료 (Phase 1~4 생략)      │
│                                                                 │
│  URGENT 분기:                                                   │
│  Phase 0에서 URGENT 판정 → L1+L4+L7 집중 → 나머지 1줄씩          │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### Phase별 소요 및 비중

```
┌───────────┬──────────┬────────────────────────────────┐
│ Phase      │ 비중      │ 핵심                           │
├───────────┼──────────┼────────────────────────────────┤
│ Phase 0    │ 15%      │ 분석의 방향을 결정. 가장 중요.  │
│ Phase 1    │ 10%      │ 데이터 수집. 도구 의존.         │
│ Phase 2    │ 40%      │ 7-Layer 해부. 분석의 본체.      │
│ Phase 2.5  │ 15%      │ 되먹임. 분석 품질 결정.         │
│ Phase 3    │ 10%      │ 돌발 질문. 창발적 사고.         │
│ Phase 4    │ 10%      │ 불확실성 고백. 자기 인식.       │
└───────────┴──────────┴────────────────────────────────┘
```

## 5. 진화 루프 (Evolution Loop)

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│  ① 분석 실행                                                     │
│     ↓                                                           │
│  ② 셀프검증 체크리스트 (errors.md 12항목)                         │
│     ↓                                                           │
│  ③ 실패 항목 → 보강 후 재출력                                     │
│     ↓                                                           │
│  ④ 인지 오류 탐지 → R-01~R-06 점검                                │
│     ↓                                                           │
│  ⑤ 오류 발견 시 errors.md 기록                                    │
│     (발견 상황 → 원인 → 영향 → 재발 방지 → 상태)                   │
│     ↓                                                           │
│  ⑥ 재발 방지에서 새 체크리스트 항목 추출                           │
│     ↓                                                           │
│  ⑦ 같은 오류 3회+ 반복 → SKILL.md 또는 가이드 수정 검토            │
│     (Yellow Zone — 사용자 확인 필요)                               │
│                                                                 │
│  이 루프가 시스템의 유일한 진화 경로.                               │
│  지침(MD)이 바뀌면 시스템이 바뀐다.                                │
│  코드는 출력만 담당하므로 진화의 대상이 아니다.                     │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 7. 다른 시스템과의 관계

```
┌──────────────────┐     L7 투자 함의      ┌──────────────────┐
│ Stereo Analyzer  │ ──────────────────→ │ PSF-Monitor      │
│ (해부)           │     PSF Property    │ (계기판)          │
│                  │ ←────────────────── │                  │
└──────────────────┘     국면/축 맥락      └──────────────────┘
        │
        │ L7 결론                        ┌──────────────────┐
        └──────────────────────────────→ │ Verification     │
                  검증 입력               │ Engine (검증)    │
                                         └──────────────────┘
        │
        │ MACRO 유형 분석 시              ┌──────────────────┐
        └──────────────────────────────→ │ Macro            │
                  지표 데이터 읽기        │ (수집자)          │
                                         └──────────────────┘

규칙:
  - Stereo ≠ PSF: 독립 스킬. PSF가 관측이면 Stereo는 해부.
  - Stereo → PSF: PSF연결 모드에서만 교차 참조.
  - Stereo → Verification: L7 결론을 6Layer 검증 입력으로 전달 가능.
  - Stereo → Macro: MACRO 유형 분석 시 데이터 참조. 직접 수집하지 않음.
```
