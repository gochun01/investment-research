# Issue Scanner 작동 원리

> 글로벌 미디어를 4축으로 스캔 → 5-Gate로 쟁점 필터 → 근인 클러스터링 → 사용자 선택 → reaction-monitor 전달

---

## 왜 4축인가

```
매일 글로벌 미디어는 수천 건의 뉴스를 생산한다.
이 중 투자에 영향을 주는 쟁점은 5~10건.

문제: "중요한 것"이 어디에 숨어 있는지 모른다.

A축만 보면: 이미 움직인 것만 봄. 선행 신호 놓침.
B축만 보면: 정책만 봄. 공급 충격 놓침.
C축만 보면: 구조만 봄. 당장 급변하는 것 놓침.
D축만 보면: 과거만 봄. 새 이슈 놓침.

4개 축이 교차하면:
┌──────────────────────────────────────────────┐
│                                                │
│     A(즉시)          B(곧)                     │
│     "이미 움직인 것"   "곧 움직일 것"           │
│          ↑               ↑                     │
│      가시성 높음      가시성 중간               │
│                                                │
│     C(느리게)         D(묻힌)                  │
│     "느리게 번질 것"   "묻혀 있는 것"           │
│          ↑               ↑                     │
│      가시성 낮음      가시성 최저               │
│                                                │
│     시간: 즉시 ──────────────────── 수개월      │
│     가시성: 높음 ────────────────── 낮음        │
└──────────────────────────────────────────────┘

4축 = 시간축(즉시→구조) × 가시성축(표면→매몰)의 2×2 매트릭스.
```

## 왜 5-Gate인가

```
"뉴스"와 "쟁점"은 다르다.

뉴스: "트럼프가 이란에 휴전안을 보냈다"
  → 사실이지만, 이것만으로는 "왜 봐야 하는가"를 모른다.

쟁점: "시장은 휴전을 프라이싱($100↓)하는데 이란은 거부+공격 중.
       호르무즈 80% 감소는 지속. 가격과 현실의 괴리."
  → 긴장이 있고, 방향이 있고, 전파가 있다.

5-Gate가 하는 일:
  G1: 이것이 시장에 닿는가? (접촉면)
  G2: 이것이 새로운 힘인가? (변화 벡터)
  G3: 이것이 번지는가? (전파 가능성)
  G4: 이것이 오늘 봐야 하는가? (긴급성)
  G5: 이것이 이미 추적 중인가? (중복)

G1 + G4 + G5 = 체크리스트 (재현성)
G2 + G3 = LLM 자유 추론 (판별력)

이 조합이 "재현성과 판별력의 균형"을 만든다.
```

## 왜 근인 클러스터링인가

```
키워드 클러스터링의 문제:
  "이란 휴전" ←→ "이란 핵" → 같은 클러스터 ✅
  "호르무즈 LNG" ←→ "비료 부족" → 다른 클러스터 ❌ (실은 같은 뿌리)

  키워드로는 "이란 휴전"과 "비료 부족"이 같은 사건의 산물인지 모른다.

근인(root cause) 클러스터링:
  이란 휴전 ────┐
  호르무즈 LNG ──┼── 근인: 미-이란 전쟁
  걸프 공습 ────┘
  비료 부족 ────── 근인: 미-이란 전쟁 (호르무즈 경유)

  → 전부 1개 클러스터.

LLM이 3가지 질문으로 판정:
  ① 같은 인과 체인? (A→B→C면 하나)
  ② 같은 지정학 사건 파생?
  ③ 같은 정책/규제 변화 파생?
  하나라도 YES → 같은 클러스터.
```

## 사각지대 보정이 하는 일

```
3-Pass 추출 엔진이 완벽하지 않다. 편향이 생긴다.

편향 1: dominant event 압도
  전쟁 기간에는 A·B·C축 모두 전쟁만 반환.
  → Q4(다양성)가 비전쟁 이슈를 최소 1건 확보.

편향 2: 자산군 편향
  오늘 원유·주식만 움직이면 채권·부동산·크립토가 빠짐.
  → Q1(자산군)이 빠진 군을 보충 스캔.

편향 3: 지역 편향
  글로벌 위기 시 한국 이슈가 묻힘 (KRW 1,500 같은 것).
  → Q2(지역)가 한쪽 0건이면 보충.

편향 4: 반복 무시
  매번 탈락하지만 계속 등장하는 이슈 = 느린 구조 변화.
  → Q3(backlog)가 3회+ 반복 이슈를 승격 후보로 표시.
```

## reaction-monitor와의 연결

```
scanner가 하는 것:
  읽기: rm/state.json, rm/active-watches.json (중복 체크용)
  전달: rm Phase 1 입력으로 이슈 제목 + 보조 컨텍스트

scanner가 하지 않는 것:
  rm의 state.json 직접 수정 ← Red Zone
  rm의 Watch 직접 등록 ← Red Zone
  rm의 Phase 0~6 실행 중 간섭 ← 금지

관계:
  scanner = "오늘 뭘 봐야 하나" (발견)
  rm = "그것에 시장이 어떻게 반응했나" (수집)

  scanner가 없으면: 사용자가 직접 쟁점을 입력 (기존 방식)
  scanner가 있으면: 스캔 → 목록 → 선택 → rm 자동 전달
```
