# 판정 규칙표 — 정량 임계값

이 문서는 모든 ✓/✗ 판정의 **최종 권위**다.
SKILL-macro-framework.md의 정성적 규칙을 정량화한 것.

> **v2 참고선 원칙**: 아래 임계값은 참고선(reference line)이다. 절대 기준이 아니다.
> 맥락에 따라 같은 수치도 다르게 판정할 수 있다.
> 판정의 근거를 반드시 서술한다. "B1 ✗"만이 아니라 "B1 ✗ — 1.88%, 4주 +8bp, 횡보" 서술.

## 1. Layer B 위험자산 방향 판정

각 지표의 ✓(위험자산 방향) / ✗(안전자산 방향) 기준:

| ID | 지표 | ✓ 조건 | ✗ 조건 | 판정 방법 |
|----|------|--------|--------|----------|
| B1 | 실질금리 | 4주간 −10bp 이상 하락 | 4주간 횡보 또는 상승 | 4주 변화량 기준. 절대 수준 아님. |
| B2 | DXY | 4주 MA 하락 추세 | 4주 MA 횡보 또는 상승 | 이동평균 방향. |
| B3 | USD/JPY | 주간 변동 ±3% 미만 | 주간 ±3% 이상 급변 | 급변 = Override. 안정 = ✓. |
| B4 | Net Liq | 4주 MA 상승 전환 | 4주 MA 하락 | 이동평균 방향. 절대량 아님. |
| B5 | HY OAS | 350bp 미만 + 4주 축소 추세 | 350bp 이상 또는 확대 추세 | 수준 + 방향 모두 충족. |

```
위험자산 방향 수 합산:
  5/5 또는 4/5 → 🟢 팽창기
  2/5 또는 3/5 → 🟡 Transition
  0/5 또는 1/5 → 🔴 수축기
```

### 경계 사례 처리

```
"거의 ✓" (예: B1이 −8bp 하락, 기준 −10bp):
  → △ (중립) 처리. ✓에도 ✗에도 넣지 않음.
  → 합산에서 0.5로 계산.
  → 보고서에 "B1 중립 (−8bp, 기준 미달)" 명시.

4/5 달성 but 교란 경로 활성:
  → 기계적으로는 🟢 팽창기.
  → 단, 보고서에 "교란 [X] 활성으로 팽창기 확정 유보" 주석 가능.
  → 레짐 자체는 규칙대로 🟢로 기록.
```

## 2. L7/L8 리스크 게이트

### L7 공식 + 정규화

```
L7 = 0.30×HY_n + 0.25×VIX_n + 0.20×SOFR_n + 0.15×MOVE_n + 0.10×TED_n

정규화 (클리핑 [0, 1]):
  HY_n   = (HY_OAS − 250) / (800 − 250)      정상: 250bp, 위기: 800bp
  VIX_n  = (VIX − 12) / (45 − 12)             정상: 12, 위기: 45
  SOFR_n = |SOFR − DFF| / 50bp                 정상: 0bp, 위기: 50bp
  MOVE_n = (MOVE − 80) / (200 − 80)            정상: 80, 위기: 200
  TED_n  = (TED − 10) / (100 − 10)             정상: 10bp, 위기: 100bp
```

### L8 공식

```
L8 = 0.35×은행CDS_n + 0.30×TED_n + 0.20×유동성동결_n + 0.15×긴급FOMC_n

데이터 가용하지 않은 항목: 0으로 처리 + "⚫ 미수집" 태깅
```

### 게이트 판정

| L7 | L8 | 판정 | 행동 |
|----|----|----|------|
| < 0.60 | < 0.60 | 🟢 정상 | 레짐 판정 진행 |
| ≥ 0.60 | < 0.60 | 🟡 경계 | 위험자산 축소. L8 집중 감시. |
| ≥ 0.60 | ≥ 0.60 | 🔴 위기 | 현금 극대화. 정책 대응 후 재진입. |

```
🔴 위기 시: 레짐 판정 불필요. 위기 보고서만 발행. 이하 단계 생략.
```

## 5. 교란 경로 활성 판정

| 교란 | 활성 조건 | 부분 활성 | 해소 조건 |
|------|----------|----------|----------|
| 교란 1 (엔캐리) | B3 주간 ±3% **+** VIX ≥ 25 동시 | B3 ±3% 단독 (VIX 미반응) | B3 안정 + VIX < 20 복귀 |
| 교란 2 (유가) | C7 주간 +15% **+** C8 +10bp 동시 | C7 +15% 단독 (C8 미반응) | C7 4주 전 수준 복귀 |
| 교란 3 (재정) | D8 +30bp **+** B1 +20bp 동시 | D8 +30bp 단독 | D8 4주 전 수준 복귀 |

```
부분 활성: 보고서에 "교란 [X] 부분 활성 (🟡)" 기록.
완전 활성: "교란 [X] 활성 (🔴)" 기록.
다중 교란 동시 발생 시:
  → 각각 독립 판정. 합산하지 않음.
  → 우선순위: 교란 1(Override) > 교란 2(역전) > 교란 3(상충)
  → 교란 1 활성 시 교란 2·3의 의미가 감소 (동반 매도가 모든 것을 지배)
```

## 6. 레짐 판정 순서 (의사결정 트리)

```
Step 1: L7 ≥ 0.60?
  ├── Yes → L8 ≥ 0.60?
  │   ├── Yes → 🔴 위기. 중단.
  │   └── No  → 🟡 경계. 축소 모드.
  └── No  → Step 2로.

Step 2: Layer B 위험자산 방향 수 계산 (위 표 기준)
  ├── 4+ → Step 3으로.
  ├── 2~3 → 🟡 Transition. 방향 미확정.
  └── 0~1 → 🔴 수축기.

Step 3: 과열 신호 확인
  ├── VIX < 12 AND HY OAS < 250bp → 🟡 과열기.
  └── 아니면 → 🟢 팽창기.

Step 4: 교란 경로 확인
  → 교란 활성 시 보고서에 주석 추가.
  → 레짐 자체는 Step 2~3 결과 유지 (교란이 레짐을 바꾸지 않음).
  → 단, "교란 활성으로 팽창기 확정 유보" 서술 가능.
```

## 7. 데이터 신선도 기준

| 지표 유형 | 최대 허용 지연 | 초과 시 |
|----------|-------------|---------|
| FRED 일간 (B1, B5 등) | T+1 영업일 | ⚠️ 플래그 + 직전값 사용 |
| FRED 주간 (WALCL 등) | T+5 영업일 | 정상 (주간 발표) |
| FRED 월간 (A1, C6 등) | 발표 후 30일 | 정상 (월간 지연) |
| Yahoo 일간 | T+1 영업일 | ⚠️ 플래그 + 웹검색 폴백 |
| 웹검색 (A2, C5 등) | T+7일 | ⚫ 미수집 태깅 + 직전값 유지 |
| AUX FRED (X1~X8) | T+1 영업일 | ⚠️ 플래그 + 직전값. PSF에 "[stale]" 전파. |
| AUX Yahoo (X9~X18) | T+1 영업일 | ⚠️ 플래그 + 직전값. PSF에 "[stale]" 전파. |
| AUX CoinGecko (X19) | T+1일 | ⚠️ 플래그 + 직전값. |
| AUX 시계열 (VIX 3~5일 등) | 당일 | 수집 실패 시 Link 트리거 판정 불가. "미확인" 태깅. |

```
데이터 충돌 시 (FRED vs 웹검색 값 불일치):
  → MCP 직접 조회값 우선 (FRED > Yahoo > 웹검색)
  → 차이 ±2% 이내: MCP값 채택, 불일치 미기록
  → 차이 ±2% 초과: MCP값 채택 + ⚠️ "값 불일치" 플래그
```

## 8. 내러티브 판정

| 조건 | 판정 | 의미 |
|------|------|------|
| 강한 경제지표 발표 → 주가 ↑ | Good news = Good | 성장 기대 > 긴축 우려 |
| 강한 경제지표 발표 → 주가 ↓ | Good news = Bad | 긴축 우려 > 성장 기대 |
| 판별 불가 (혼조) | 유지 | 전주 판정 유지 |

