# 운영 사이클 — OPERATIONS

## 목적
매크로 시스템의 정기/비정기 운영 일정과 절차를 한 곳에 정의한다.

## 주간 사이클 (매주 월요일)

### 실행: `/macro-weekly`

```
소요 시간: ~10분
산출물:
  ├── indicators/YYYY-MM-DD.yaml (데이터 스냅샷)
  ├── reports/YYYY-MM-DD_macro-weekly.md (보고서)
  └── Notion DB ee345e95 (아카이브)

절차:
  1. 직전 주 indicators/ YAML 로드 (비교 기준)
  2. PLUGIN-weekly-macro.md Step 1~10 실행
  3. indicators/ 에 신규 YAML 저장
  4. 전주 대비 diff 생성 → 보고서 "전주 대비" 섹션에 반영
  5. reports/ 에 보고서 저장
  6. Notion 아카이브
```

### 사전 체크리스트

```
□ MCP 서버 연결 확인 (FRED, Yahoo)
□ 전주 보고서/YAML 존재 확인
□ 이번 주 예정 이벤트 확인 (FOMC, CPI, 고용 등)
□ 전주 트리거 발동 여부 확인 → 해소/지속 판정
```

### 산출물 검증

```
□ indicators/ YAML 27개 지표 전부 채워짐
□ confidence: black (미수집) 지표 3개 이하
□ 레짐 판정과 L7/L8 정합
□ 보고서 형식 TEMPLATE 준수
□ Notion 아카이브 완료
```

## 월간 사이클 (매월 첫째 주 월요일, 주간과 함께)

### 추가 실행 항목

```
1. 전제 점검 (SKILL-macro-verification.md)
   → P1~P5 전제 유효성 재확인
   → 전월 판정 vs 실제 시장 역추적
   → 전제 수정 필요 시 verification 문서 갱신

2. 트리거 임계값 리뷰 (triggers.md)
   → 전월 발동된 트리거 목록
   → False positive / False negative 분석
   → 임계값 조정 필요 시 triggers.md 갱신

3. 지표 스키마 리뷰 (indicators/_schema.md)
   → 신규 지표 추가 필요 여부
   → 폐기 지표 여부
   → 소스 변경 여부 (MCP 엔드포인트 등)

4. indicators/ 이력 분석
   → 4주 추세 요약 (방향 전환 식별)
   → 레짐 전환 이력
   → 키스톤 상태 변화
```

### 역추적 검증 (월간 핵심)

```
전월 판정이 실제 시장과 맞았는가:

[YYYY-MM 역추적]
├── 전월 레짐 판정: [🟢/🟡/🔴]
├── 실제 시장 결과: S&P 500 [+X%/-X%], KOSPI [+X%/-X%]
├── 판정 적중 여부: [적중/부분적중/미적중]
├── 미적중 시 원인:
│   ├── 데이터 오류 (Layer 1)?
│   ├── 추론 오류 (Layer 2)?
│   └── 전제 붕괴 (Layer 3)?
├── 교란 경로 발동 이력:
│   ├── 발동일 + 조건 + 결과
│   └── False positive 여부
├── 트리거 성과:
│   ├── 발동 [N]회, 적중 [N]회, 오경보 [N]회
│   └── 임계값 조정 필요 여부
└── 조치: [임계값 변경 / 전제 수정 / 없음]
```

### 산출물

```
├── reports/YYYY-MM_monthly-review.md (역추적 포함)
├── SKILL-macro-verification.md 갱신 (필요 시)
├── RULES.md 갱신 (임계값 조정 시)
└── triggers.md 갱신 (필요 시)
```
