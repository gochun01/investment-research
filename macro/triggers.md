# 비정기 실행 트리거 — 통합 규칙서

## 목적
`/macro-weekly` 외에 즉시 실행이 필요한 상황을 정의한다.
트리거 발동 시 전체 10-Step이 아닌 **최소 필수 Step만** 실행하여 속도를 확보한다.

---

## 트리거 유형

### Type 1: 지표 급변 트리거

| 트리거 | 조건 | 실행 범위 | 긴급도 |
|--------|------|----------|--------|
| VIX 스파이크 | VIX 30+ 돌파 | Step 1(L7/L8) → 위기 시 중단 보고서 | 🔴 즉시 |
| 엔캐리 경보 | USD/JPY 주간 ±3% | Step 1 + Step 3(B3) + 교란 1 점검 | 🔴 즉시 |
| 유가 급등 | Brent 주간 +15% | Step 5(C7,C8) + 교란 2 점검 | 🟡 당일 |
| 유가 급락 | Brent 주간 -15% | Step 5(C7,C8) + 교란 2 해소 확인 | 🟡 당일 |
| HY 확대 | HY OAS +100bp | Step 1(L7) + Step 3(B5) | 🔴 즉시 |
| DXY 급변 | DXY 주간 ±3% | Step 3(B2) + 경로 재판정 | 🟡 당일 |

### Type 2: 이벤트 트리거

| 트리거 | 조건 | 실행 범위 | 긴급도 |

| FOMC 결정 | 금리 변경 또는 가이던스 변화 | Step 2(A1) + Step 3(B1,B4) + Step 7 레짐 재판정 | 🟡 당일 |

## 트리거 실행 프로토콜

### 발동 절차

```
1. 트리거 조건 감지
   → 사용자 보고 또는 뉴스/데이터 확인 시

2. 해당 트리거의 실행 범위 확인 (위 표)

3. MCP 데이터 수집 (해당 지표만)

4. indicators/ 에 트리거 스냅샷 저장
   파일명: YYYY-MM-DD_trigger_[이벤트명].yaml

5. 최신 주간 YAML과 비교 → 변동분 식별

6. 경로 재판정
   → 교란 경로 활성/비활성 판정
   → 레짐 변경 필요 여부

7. 보고서 생성 (축약 형식)
   → 1페이지 요약 + 변동분만
   파일명: YYYY-MM-DD_trigger_[이벤트명].md
```

### 축약 보고서 형식

```markdown
# [트리거명] 긴급 점검 (YYYY-MM-DD)

## 트리거
[조건] → [발동 사유]

## 변동 지표
| 지표 | 직전값 | 현재값 | 변화 | 의미 |

## 경로 영향
[활성/비활성 경로] → [레짐 영향]

## 행동 함의
[즉시 조치 사항]

## 다음 체크
[모니터링 포인트]
```

---

## 트리거 이력 관리

```
모든 트리거 발동 기록:
  indicators/YYYY-MM-DD_trigger_[이벤트명].yaml

트리거 해소 기록:
  다음 주간 보고서에 "전주 트리거 [X] → 해소/지속" 명시

월간 리뷰:
  발동된 트리거 vs 실제 시장 영향 역추적
  → 임계값 조정 필요 시 이 문서 갱신
```
