## Cognitive Error Defense (R-01 ~ R-06)

macro 분석에서 반복되는 인지 오류 패턴과 방어 규칙.

### R-01: 레짐 관성

```
패턴: 전주와 같다고 가정. "변화 없음"으로 넘어감.
위험: 점진적 변화 누적을 놓침. 특히 B1 횡보→하락 전환 시점.
방어: 레짐이 전주와 같을 때 반드시 "유지 근거"를 재검증한다.
      "같다"는 결론이 아니라 "왜 같은가"를 서술한다.
검사: Q4(진화규칙)에서 관성 방어 여부 확인.
```

### R-02: 키스톤 과신

```
패턴: Core PCE만 보고 전체 레짐을 판단. B1~B5 전수 확인 생략.
위험: A1이 안정적이어도 B3(엔캐리) 급변 시 Override 발생.
방어: A1 판정 후 반드시 Layer B 전수 판정으로 이동.
      A1 안정 ≠ 레짐 안정. 전달 채널(B)이 끊어질 수 있다.
검사: V-03(B1~B5 전수 판정) 확인.
```

### R-03: 이벤트 지배

```
패턴: 큰 이벤트(전쟁, FOMC 서프라이즈)가 수치 판정을 덮음.
위험: "이란전 때문에 수축기"처럼 이벤트가 레짐을 직접 결정.
방어: 이벤트는 경로(path)를 통해서만 레짐에 영향.
      "이벤트 → 어떤 지표 → 어떤 변화 → 레짐"을 서술.
      이벤트가 지표를 바꾸지 않았으면 레짐은 변하지 않는다.
검사: V-04(인과 체인 데이터 근거) 확인.
```

### R-04: 은유 확정

```
패턴: "경로가 포위됨", "물이 막힘", "시장이 베팅 중" 같은 은유를 확정처럼 사용.
위험: 은유가 분석을 대체. 데이터 없이 서사가 주도.
방어: CLAUDE.md §인과추적의범위 기준 적용.
      "이 문장에서 은유/의인화를 제거해도 의미가 전달되는가?"
      전달되면 재서술, 안 되면 삭제 또는 CE/TE 위임.
검사: V-04, V-08 확인.
```

### R-05: 보조 지표 경시

```
패턴: 핵심 27개만 보고 보조 23개(Layer AUX) 무시.
위험: PSF 매핑이 불완전. Link 트리거 시계열 미수집으로 급성 신호 누락.
방어: Step 6.5에서 AUX 전수 수집. 실패 시 ⚠️ + null + "[미수집]" 태깅.
      HYG(X11), LQD(X12) 급락은 L7/L8 급성 트리거이므로 특히 주의.
검사: V-07(보조 23개 확인) 확인.
```

### R-06: 양측 논거 생략

```
패턴: 한 방향(팽창 또는 수축)만 강조. 반대 논거를 제시하지 않음.
위험: 확증 편향. 특히 레짐 유지 판정 시 "왜 바뀌지 않는가"를 간과.
방어: 판정 전 반드시 "팽창 최강 논거 / 수축 최강 논거"를 병기.
      그 위에서 판정을 내린다.
검사: V-12(양측 최강 논거) 확인.
```

### 기록 규칙

```
1. /macro-weekly 실행 중 또는 실행 후 발견된 오류를 즉시 기록한다.
2. 유형: DATA(데이터 오류), LOGIC(판정 오류), PROCESS(절차 누락), COGNITIVE(인지 편향)
3. R-코드: 해당 Cognitive Error Defense 코드 (R-01~R-06). 해당 없으면 "—"
4. 심각도: 🔴 HIGH(레짐 판정 오류) / 🟡 MEDIUM(지표 누락/오기) / 🟢 LOW(형식 미준수)
5. 상태: OPEN(미해결) / FIXED(수정됨) / WONTFIX(구조적 한계)
```
