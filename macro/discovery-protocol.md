# macro 신호 발견 프로토콜

> macro 버전의 개방형 추론 프로토콜.
> 46개 지표를 기계적으로 채우는 것이 아니라, 변화를 발견하고 인과를 추적한다.
> L7/L8 위기 체크(Step 1)만 고정. 나머지는 데이터가 순서를 결정한다.

---

## 원칙

```
1. L7/L8 먼저, 나머지는 자유
   위기 체크만 고정 순서. 위기 아니면 "가장 크게 변한 것부터" 판다.

2. 46개를 다 채우지 않아도 된다
   변화 없는 지표를 반복 서술하지 않는다.
   변화 있는 것만 깊이. 나머지는 전수 점검에서 확인.

3. 인과 체인은 고정 경로가 아니다
   정상1/정상2/교란1/교란2/교란3은 참고 경로.
   데이터가 새 경로를 보여주면 그것을 따른다.

4. 불일치가 가장 중요하다
   B↔C 불일치, macro 레짐↔PSF 국면 불일치 — 이것이 핵심 발견.
```

---

## Step 1: L7/L8 위기 체크 (고정 — 항상 먼저)

```
FRED: BAMLH0A0HYM2, SOFR
Yahoo: ^VIX, ^MOVE
L7/L8 공식 적용 (RULES.md 참조)
🔴이면 여기서 중단. 위기 보고서 발행.
🟢/🟡이면 다음으로.
```

---

## Step 2: 변화 감지 — "전주 대비 뭐가 달라졌는가"

```
indicators/latest.json의 전주값(prev)과 이번 주 수집값을 비교.

질문:
  ① 가장 크게 변한 Layer B 지표는? (절대값이 아니라 평소 대비)
  ② Layer C에서 B와 불일치하는 것이 새로 생겼는가?
  ③ Layer D에서 평소 안 움직이던 것이 움직였는가?
  ④ 전주 이벤트 서사에서 예상한 방향과 실제가 일치하는가?

이 단계의 산출물:
  "이번 주 주목 변화 목록" — 개수는 데이터가 결정.
```

---

## Step 3: 인과 추적 — "왜 변했고, 어디로 가는가"

```
각 주목 변화에 대해:
  ① 원인 이벤트는 무엇인가? (Tavily 검색)
  ② 어떤 인과 경로를 타고 있는가?
     → 참고 경로: 정상1(Fed), 정상2(중국), 교란1(엔캐리), 교란2(유가), 교란3(재정)
     → 참고 경로에 안 맞으면 새 경로를 서술.
  ③ 이 경로가 다른 Layer에 어떤 영향을 줄 것으로 보이는가?

3~5문장 서사 작성. 서사의 길이는 변화의 크기에 비례.
```

---

## Step 4: 레짐 판정

```
Layer B 5개 위험자산 방향 수 → RULES.md 기준으로 판정.
이것은 정량 규칙이므로 유지한다.

단, 임계값은 참고선이다:
  "거의 ✓" (△)인 것은 맥락에서 판단.
  교란 경로 활성 시 레짐에 주석 부착 가능.
```

---

## Step 5: psf-monitor 연결

```
macro 레짐(🟢🟡🔴)과 PSF 국면(🟢🟡🔴)을 대조한다.
interfaces-macro.md의 정합 매트릭스 참조.
불일치 시 보고서에 명시.
```

---

## Step 6: 전수 점검

```
[Layer 커버리지]
  □ Layer A (2개) 확인했는가?
  □ Layer B (5개) 전수 수집했는가?
  □ Layer C 불일치 점검했는가?
  □ Layer D 변동 있는 것 확인했는가?
  □ Layer AUX (PSF용 보조) 수집했는가?

[인과 일관성]
  □ 이벤트 서사와 레짐 판정이 일관적인가?
  □ B↔C 불일치를 모두 설명했는가?

[이력 연속성]
  □ 전주 보고서의 "다음 확인" 항목에 답했는가?
  □ 전주 예상과 실제가 일치/불일치하는지 명시했는가?

[관측/해석 분리]
  □ "~에 베팅", "~해야 한다" 같은 해석 표현이 없는가?
  □ 판정 근거가 수치로 뒷받침되는가?
```

---

## 산출물

```
보고서는 "핵심 3줄 + 해석"으로 시작한다.

핵심 3줄:
  1줄: 레짐 + 변화 방향 (유동성 렌즈)
  2줄: 가장 중요한 인과 체인 (이벤트 서사)
  3줄: 다음 주 최우선 감시 (분기점)

해석:
  각 줄을 2~3문장으로 개인투자자 수준으로 풀어쓴다.

상세:
  변화가 있는 것만. state.json 수치 재나열 금지.

저장:
  reports/YYYY-MM-DD_macro-weekly.md
  indicators/latest.json 갱신
  Notion 아카이브

PM_핵심예측 작성 규칙 (LOOP 1 연동): ★ 개선
  예측은 반드시 검증 가능해야 한다. 모호한 서술 금지.

  금지:
    "🟡 Transition 유지" ← 범위가 없음. 어떻게 되면 틀린 건지 모름.
    "교란 지속" ← 언제까지? 어느 수준까지?

  필수:
    수치 범위 + 기한이 있어야 한다.
    "3/30까지 USD/JPY 155~163 레인지. 이탈 시 오판."
    "3/30까지 B5 HY OAS 350bp 미돌파."
    "4/1 ISM PMI 50 이상 유지."

  구조:
    PM_핵심예측 = [기한] + [지표] + [범위 또는 방향] + [이탈 시 판정]
    PM_검증일 = 기한과 동일 날짜
```
