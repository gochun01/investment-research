# 핵심 이슈 검증기 v2.0

> scanner가 "뭘 봐야 하나"를 발견하면, core-extractor가 "이것이 진짜인가"를 검증한다.
> scanner에 없는 4가지만 한다: 다름 검증, 제외 검색, 느린 변화, 주간 복기.

---

## scanner와의 관계 — 병렬 구조

```
scanner(발견)와 core-extractor(검증)는 직렬이 아니라 병렬이다.
둘 다 독립적으로 실행되고, 결과가 rm에 합류한다.

scanner가 하는 것:              core-extractor가 하는 것:
━━━━━━━━━━━━━━━━              ━━━━━━━━━━━━━━━━━━━━━
4축 스캔 (A/B/C/D)              3문 테스트 (D-1/D-2/D-3)
5-Gate 점수                     제외 검색 (묻힌 것)
클러스터링                       메가트렌드 축 연결 (MT-01~07)
backlog                         느린 변화 (3개월 추세)
[CUMULATIVE] 태그               주간 복기 (맞았는가?)
scanner_prefetch → rm           core_extract → rm

겹치지 않는다. 각자 고유한 것만 한다.

흐름:
  1) scanner ──→ scanner_prefetch ──┐
                                    ├──→ rm(수집)
  2) core-extractor ──→ core_extract ──┘

  scanner가 "어떤 이슈가 있는가"를 발견하고,
  core-extractor가 "그것이 진짜인가, 묻힌 것은 없는가"를 검증한다.
  rm은 둘 다 받아서 수집한다.

  scanner 없이 core-extractor만 독립 실행도 가능.
  core-extractor 없이 scanner→rm도 가능.
  둘 다 있으면 rm의 입력이 가장 풍부해진다.
```

---

## 입력

```
A. scanner 결과를 받는 경우 (권장):
   scanner Phase S5에서 사용자가 선택한 이슈/클러스터
   → core-extractor가 "이것이 진짜인가" 3문 테스트
   → 통과한 것만 rm에 전달

B. 독립 실행:
   사용자가 직접 이슈/기사 URL/키워드를 제공
   → core-extractor가 전체 프로세스 실행
   → rm에 전달
```

---

## 실행 흐름

### Step 1: 3문 테스트 — "이것이 진짜인가" (3분)

scanner에서 온 이슈 또는 직접 입력 이슈에 3개 질문:

```
① 어제와 다른가?
   D-1: 새로운 수치가 있는가?
     예: "PPI 0.7% (예상 0.3%)" — 어제 없던 수치.
   D-2: 새로운 주체가 등장했는가?
     예: "파키스탄이 중재국으로 부상" — 어제 없던 주체.
   D-3: 방향이 바뀌었는가?
     예: "이란, 비적성 선박 통행 허용" — 기존 봉쇄에서 반전.

   셋 다 없으면 → "반복". 버림. rm 전달 안 함.

② 수치가 있는가?
   구체적 숫자(%, $, bp, 건 등)가 있는가?
   NO면 → "서사". 버림.
   예외: 정책 발표(법안, 성명)는 수치 없어도 통과.

③ 나 혼자만 보는가?
   모든 매체가 똑같이 쓰는 기사인가?
   YES(동조) → 버림.
   단, 한 나라만 보도하는 것은 "묻힌 것"으로 살림.
```

**산출**: 통과 이슈에 `[D-1]` `[D-2]` `[D-3]` 태그 부착.

### Step 2: 제외 검색 — "묻힌 것 찾기" (2분)

```
scanner의 사각지대 Q4와 다른 방법:
  Q4는 "결과를 보고 다양성 부족 시 보충"
  제외 검색은 "지배적 키워드를 빼고 처음부터 다시 검색"

방법:
  오늘 모든 매체의 톱 기사 → 지배적 키워드 2~3개 추출
  WebSearch: "-[키워드1] -[키워드2] latest economic market news today"
  → 결과에서 3문 테스트 통과하는 것 1~2건 추출

  예: 오늘 지배적 = "Iran war ceasefire"
  → "-Iran -war -ceasefire latest economic news March 2026"
  → "독일 제조업 PMI 20개월 최저" 발견 — 전쟁에 묻힘

산출: 묻힌 것 0~2건. [BURIED] 태그.
```

### Step 3: 메가트렌드 축 연결 — "그래서 뭐" (3분)

```
통과한 이슈를 메가트렌드에 꿰맞춤:

축 목록:
  MT-01: AI/반도체 capex 사이클
  MT-02: 에너지 전환 + 지정학
  MT-03: 고령화/인구 구조
  MT-04: 블록체인/크립토 제도화
  MT-05: 재정지배/금융억압
  MT-06: 미중 디커플링
  MT-07: 글로벌 공급망 재편

꿰기 질문:
  ├── 어느 축에 닿는가?
  ├── 교차점에 닿는가? (MT-01×MT-02 등)
  ├── 서로 충돌하는 이슈가 있는가?
  └── 아무 축에도 안 닿으면 → "잡음 또는 신규 축 후보"

꿰기 규칙:
  같은 축 3건+ → "이 축에서 뭔가 움직이고 있다"
  2개 축 동시 → 교차점 활성화 신호
  어디에도 안 걸림 → 판단 보류 + backlog 후보

산출: 각 이슈에 [MT-XX] 또는 [MT-XX×MT-YY] 태그.
```

### Step 4: 느린 변화 점검 — 주 1회 월요일 (2분)

```
3문 테스트는 날카로운 변화를 잡고,
이 점검은 둔탁한 변화를 잡는다.

질문: "3개월 전과 비교해서 방향이 달라진 것이 있는가?"

점검 대상 (축당 핵심 수치 1개):
  MT-01: AI capex 추이 (분기별)
  MT-02: Brent 유가 3M 추세 (macro C7 trend_note 참조)
  MT-03: 주요국 고용지표
  MT-04: 스테이블코인 총발행량 / ETF AUM
  MT-05: 미국 부채/GDP, 실질금리 (macro B1 trend_note 참조)
  MT-06: 미중 관세 강도
  MT-07: 글로벌 교역량

판정:
  같은 방향 유지 → "추세 건재" — 넘어감
  가속 → "추세 강화" — 주목
  감속 → "추세 둔화" — 경고
  반전 → "방향 전환" — 최우선 [SLOW-SHIFT] 태그

산출: 느린 변화 0~2건. [SLOW-SHIFT] 태그.

★ macro의 trend_note를 참조한다 (직접 수집 안 함).
```

### Step 5: rm 전달 (패스스루 포함)

```
core-extractor 산출물을 rm에 전달:

전달 형식:
  core_extract: {
    tested_issues: [
      {title, d_tag: "D-1|D-2|D-3", mt_tag: "MT-XX", fact: "1문장+수치"}
    ],
    buried: [
      {title, source, reason, mt_tag}
    ],
    slow_shifts: [
      {axis: "MT-XX", direction: "가속|감속|반전", evidence: "3M 수치 비교"}
    ],
    noise_summary: "오늘 지배적 소음: [주제]. 새로운 팩트 없음."
  }

rm은 core_extract를 받으면:
  tested_issues → Phase 1 지문의 보조 컨텍스트
  buried → Phase 2 채널 선정에 추가 채널 후보
  slow_shifts → Phase 4 패턴 판독에 장기 맥락
  → scanner_prefetch와 별도. 둘 다 있으면 둘 다 사용.

rm이 core_extract 없이도 동작:
  기존대로 전체 수집. 변화 없음.
```

## 주간 복기 — 금요일 (3분)

```
질문: "지난주 핵심이 맞았는가?"

실행:
  지난주 tested_issues 꺼내기
  각 건에 대해:
    시장이 반응했는가? → ✅ 맞음 / ❌ 틀림 / ⏳ 미확인
    맞았으면 → D-1/D-2/D-3 중 어떤 기준이 적중?
    틀렸으면 → 왜? (소음이었는가, 시간이 안 됐는가)

  묻힌 것(buried)이 표면화했는가?
    YES → 제외 검색 작동 확인
    NO → 정말 잡음이었거나 아직 시간 필요

  교정 메모:
    12주 축적 후 → 3문 테스트 기준 자체를 교정
    D-1/D-2/D-3 적중률 비교 → 가장 유효한 기준 강화

산출: 복기 카드 (✅/❌/⏳ + 교정 메모)
```

## 파이프라인 위치

```
scanner(발견) → core-extractor(검증+보강) → rm(수집) → Stereo(해부)
                     │
                     ├── 3문 테스트: "이것이 진짜인가"
                     ├── 제외 검색: "묻힌 것은 없는가"
                     ├── 축 연결: "어디에 닿는가"
                     └── 느린 변화: "둔탁한 것은 없는가"

scanner가 없어도 독립 실행 가능:
  사용자 입력 → core-extractor → rm
```

## 실패 패턴 방어

```
F-01: 3문 테스트 없이 직감으로 판단 → 반드시 D-1/D-2/D-3 명시
F-02: 제외 검색 건너뛰기 → Standard 모드에서도 반드시 1회
F-03: 축에 안 꿰고 팩트만 나열 → Step 3 없으면 뉴스 클리핑
F-04: 느린 변화 누락 (월요일) → 모드 자동 전환
F-05: 복기 누락 (금요일) → 모드 자동 전환
F-06: 묻힌 이슈 0건 → 제외 검색 재확인
F-07: 핵심 6건+ → 3~5건 압축. 우선순위가 없는 것
F-08: macro trend_note 미참조 (느린 변화) → macro 데이터 필수 확인
```
