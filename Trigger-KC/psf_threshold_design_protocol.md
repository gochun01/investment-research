# PSF 임계값 설계 및 재조정 프로토콜

> **핵심 원칙**: 임계값은 "숫자를 정하는 것"이 아니라 "행동이 바뀌는 지점을 식별하고, 그 지점에 3중 밴드를 설치하고, 구조가 바뀌면 밴드를 재보정하는 프로토콜"을 만드는 것이다. 숫자는 바뀌어도 되지만, 이 프로토콜은 바뀌면 안 된다.

---

## 1. 임계값의 본질

```
임계값은 "숫자"가 아니라 "상태 전환점"이다
│
├── 잘못된 설계: "Brent $100을 임계값으로 설정하자"
│   └── 왜 $100인가? $99와 $101의 질적 차이는 무엇인가?
│
├── 올바른 설계: "이 숫자를 넘으면 시장 참여자의 행동이 바뀌는 지점"
│   └── $100 = 옵션 행사가 집중 + 심리적 앵커 + 미디어 헤드라인 전환
│   └── 즉, $100 자체가 중요한 게 아니라 "$100에서 행동이 바뀐다"가 중요
│
└── 정의: 임계값 = 가격이 아니라 "행동 전환점(behavioral tipping point)"
```

---

## 2. 임계값의 3가지 원천 (Source)

### Source 1: 구조적 임계값 — 시장 메커니즘에서 도출

```
시장의 물리적·재무적 구조가 강제하는 전환점

├── 옵션 스트라이크 집중 구간
│   └── 대량 행사가 = 시장 참여자의 "방어선"
│
├── 마진콜 수준
│   └── 레버리지 포지션이 강제 청산되는 가격
│
├── 손익분기점
│   └── 생산자의 shut-in 임계값 (예: 셰일 $55~65)
│
├── 저장 용량 한계
│   └── 물리적으로 더 이상 담을 수 없는 지점
│
└── 특징: 객관적, 측정 가능, 비교적 안정적
```

**적용 예시:**

| 임계값 | 수준 | 근거 |
|--------|------|------|
| Brent $150 | recession 임계점 | 소비자 지출의 에너지 비중이 역사적 위기 수준 도달 (BlackRock Fink 명시) |
| Crack spread $25 | 정유사 최소 운영 마진 | 이 아래면 정유소 가동 축소 → 공급 감소 자기강화 루프 |
| TTF/HH 10x | LNG 차익거래 경제성 상한 | 이 아래로 내려오면 물류 정상화 직접 증거 |

### Source 2: 통계적 임계값 — 과거 데이터에서 도출

```
역사적 분포에서 "극단"을 정의하는 전환점

├── 표준편차 기반
│   └── 2σ 이탈 = "평소에 안 일어나는 일이 일어남"
│
├── 백분위 기반
│   └── 상위/하위 5% = 극단 영역 진입
│
├── 이동평균 이탈
│   └── 20일/50일/200일 MA 대비 괴리율
│
└── 특징: 데이터 의존적, 기간 선택에 민감, 정기 재보정 필요
```

**적용 예시:**

| 임계값 | 수준 | 근거 |
|--------|------|------|
| Gold/Brent 55x | 과거 30년 상위 3% | 이 위에서는 역사적으로 항상 recession 동반 |
| OVX/VIX 2.0x | 2σ 이탈 | 에너지 변동성이 시장 전체의 2배 → 에너지 특화 위기 |
| Brent-WTI $12 | 상위 5% | 대서양 횡단 공급 단절 극단적 수준 |

### Source 3: 서사적 임계값 — 시장 참여자 합의에서 도출

```
시장 심리와 미디어가 만드는 자기실현적 전환점

├── 라운드 넘버
│   └── $100, $150 같은 심리적 앵커
│
├── 정책 발언 앵커
│   └── "Fed가 X를 언급한 수준", "Fink이 명시한 기준선"
│
├── 미디어 헤드라인 전환점
│   └── 보도 프레임이 바뀌는 가격
│
└── 특징: 주관적이지만 자기실현적(self-fulfilling)
```

**적용 예시:**

| 임계값 | 수준 | 근거 |
|--------|------|------|
| Brent $100 | de-escalation 기준선 | Fink 발언을 시장이 앵커로 채택 → 실제 전환점화 |
| 미 휘발유 $4/gal | 소비자 행동 변화 | 역사적으로 $4 돌파 시 운전 마일리지 감소 시작 |

## 3. 3중 밴드 설계 (Graduated Response)

```
단일 임계값이 아니라 3중 밴드로 설계한다

┌─────────────────────────────────────────────────┐
│                                                 │
│   ━━━━ 외곽: Hard KC ━━━━━━━━━━━━━━━━━━━━━━━━  │  ← 돌파 시 무조건 사전 정의된 행동 실행
│                                                 │
│   - - - 중간: Alert - - - - - - - - - - - - -   │  ← 진입 시 헤지 확대, 유동성 확보
│                                                 │
│   ····· 내곽: Watch ···························  │  ← 진입 시 감시 주기 상향 (일→반일)
│                                                 │
│          [현재 가격이 이 공간 안에서 움직임]     │
│                                                 │
└─────────────────────────────────────────────────┘
```

### 왜 3중인가

```
├── 단일 임계값의 문제 = "all or nothing"
│   ├── $149에서는 아무것도 안 하다가 $150에서 갑자기 전부 행동?
│   └── 현실에서 이런 이진 판단은 실행 불가
│
├── 3중 밴드의 장점 = 점진적 대응(graduated response)
│   ├── Watch: 인지적 준비 — "이 시나리오가 현실화될 수 있다"
│   ├── Alert: 행동적 준비 — 헤지 확대, 유동성 확보
│   └── Hard KC: 실행 — 사전 정의된 플랜 즉시 가동
│
└── 군사 방어 체계와 동일한 논리 (DEFCON 5→4→3→2→1)
```

### 3중 밴드에 "지속 조건"을 추가한다

```
"순간 돌파"와 "지속 돌파"를 구분해야 한다

├── 순간 돌파: 장중 $150 터치 후 $140 복귀
│   └── 이것만으로 Hard KC 발동하면 → 노이즈에 청산 → 바로 후회
│
├── 지속 돌파: $150 이상 3일 유지
│   └── 시장 구조가 실제로 그 수준을 "수용"했다는 증거
│
└── 설계 공식: [가격 수준] × [지속 일수] = 임계값
    ├── Watch: 수준 도달 (지속 조건 없음)
    ├── Alert: 수준 도달 × 3일 유지
    └── Hard KC: 수준 도달 × 3~5일 유지
```

## 4. 임계값 유형별 설계 가이드

```
┌──────────┬────────────────┬────────────────┬──────────────────┐
│ 유형     │ 절대값          │ 상대값          │ 비율/스프레드     │
├──────────┼────────────────┼────────────────┼──────────────────┤
│ 예시     │ Brent $150     │ 20일MA +40%    │ Gold/Brent 55x   │
├──────────┼────────────────┼────────────────┼──────────────────┤
│ 장점     │ 직관적          │ 체제 적응       │ 구조 변화 감지    │
│          │ 소통 용이       │ 자동 재보정     │ 인과 관계 측정    │
├──────────┼────────────────┼────────────────┼──────────────────┤
│ 약점     │ 구조 변화 둔감  │ MA 기간 의존    │ 분모 급변 시 왜곡 │
│          │ 수동 재조정     │ 후행적          │ 직관성 낮음       │
├──────────┼────────────────┼────────────────┼──────────────────┤
│ 적합     │ Hard KC         │ Watch / Alert  │ Trigger 점화     │
│ 용도     │ (최종 방어선)   │ (조기 경보)    │ (전이 확인)       │
├──────────┼────────────────┼────────────────┼──────────────────┤
│ 재조정   │ 분기 1회 검토   │ 자동(MA 이동)  │ 반기 1회 검토    │
│ 주기     │                │                │                  │
└──────────┴────────────────┴────────────────┴──────────────────┘
```

### 설계 공식

```
├── Trigger 임계값 = 비율/스프레드 우선 (전이 감지에 최적)
├── KC Watch/Alert = 상대값 우선 (체제 적응 + 조기 경보)
└── KC Hard = 절대값 + 상대값 병렬 (어느 하나라도 돌파 시 작동)
    └── 예: "Brent $150 OR 20일MA +50%, 둘 중 먼저 도달하는 것"
```

## 5. 임계값 조정(Recalibration)이 필요한 3가지 상황

### Case 1: 변동성 체제 전환 (Volatility Regime Change)

```
전쟁 전: Brent 일간 변동 ±$1~2 → 2σ 이탈 = ±$4
전쟁 후: Brent 일간 변동 ±$5~10 → 2σ 이탈 = ±$20

문제: 전쟁 전 기준 $4 이탈을 "이상 신호"로 쓰면,
      전쟁 후에는 매일 경보가 울림 → 경보 피로(alert fatigue)

해법: 변동성 체제별로 임계값 세트를 분리
├── 평시 임계값 세트 (VIX < 20)
├── 긴장 임계값 세트 (VIX 20~35)
└── 위기 임계값 세트 (VIX > 35)
→ 현재 VIX 25 = "긴장" 세트 적용 중
```

### Case 2: 구조적 수준 이동 (Level Shift)

```
전쟁 전: Brent 균형가 $66 → $100은 "위기"
전쟁 후: 새 균형가 $90~100 형성 중 → $100은 "일상"

문제: $100을 KC로 고정하면, 새 균형에서는 항상 KC 근처
      → 의미 없는 경계 상태 지속

해법: 기준점(anchor)을 절대값에서 상대값으로 전환
├── 절대값: "Brent $150" → 구조 변화에 둔감
└── 상대값: "Brent 20일MA 대비 +40%" → 구조 변화에 적응
→ 둘 다 유지하되, 상대값을 1차 필터로 사용
```

### Case 3: 인과 구조 변화 (Causal Structure Change)

```
전쟁 전: 유가 상승 → 인플레 → Fed 긴축 → 주식 하락 (정상 체인)
전쟁 후: 유가 상승 → 공급 파괴 → 수요 파괴 → 유가 자체 하락 (자기교정)

문제: 기존 인과 체인에 기반한 임계값이 새 체인에서는 무의미

해법: 임계값을 "가격"이 아니라 "전이 비율"로 설정
├── 가격 임계값: "Brent $150" → 인과 구조 변화에 둔감
└── 전이 비율 임계값: "나프타/Brent 비율 1.1x" → 인과 관계 자체를 측정
→ 비율/스프레드 기반 임계값이 단일 가격 임계값보다 구조 변화에 강건
```

## 6. 재조정(Recalibration) 프로토콜

### 정기 재조정 (Scheduled)

```
주기: 월 1회 (모닝 브리핑 사이클과 동기화)

점검 항목:
├── ① 변동성 체제 변했는가? (VIX 밴드 이동 여부)
├── ② 가격 수준 구조적으로 이동했는가? (새 균형가 형성 여부)
├── ③ 인과 체인 작동 중인가? (전이 비율이 예상대로 움직이는가)
└── ④ 경보 피로 발생했는가? (Watch/Alert가 상시 점등 상태인가)

재조정 방법:
├── 통계적: 직전 20일 데이터로 σ 재계산 → 밴드 업데이트
├── 구조적: 새 균형가 식별 → 절대값 임계값 이동
└── 서사적: 시장 앵커 변화 확인 → 심리선 업데이트

기록: 변경 전/후 값 + 변경 사유 → Notion 아카이브(DB ee345e95)
```

### 비정기 재조정 (Event-Triggered)

```
트리거: 판(板) 전환 이벤트 발생 시 즉시
├── 전쟁 개시/종료
├── 중앙은행 정책 전환 (pivot)
├── 물리적 인프라 파괴/복구
└── 새로운 시장 구조 형성 (예: 호르무즈 우회로 개설)

방법: 전체 임계값 세트 일괄 검토
├── "이 임계값이 새 구조에서도 의미가 있는가?"
└── 없으면 즉시 재설계 — 이전 값에 집착하지 않음

주의: 이벤트 직후 감정적 재조정 방지
├── 이벤트 발생 → 24시간 관찰 → 재조정 실행
└── "충격 직후 임계값을 바꾸면 패닉에 임계값을 맞추는 것"
```

### 재조정 금지 상황

```
├── ❌ 포지션이 손실 중이라서 KC를 늦추고 싶을 때
│   └── 이것은 재조정이 아니라 "손절 회피" — KC의 존재 이유 부정
│
├── ❌ 임계값 근처에서 진동 중일 때 (noise band)
│   └── 3중 밴드의 Watch가 이 상황을 흡수해야 함
│
└── ❌ 단일 이벤트에 과잉 반응할 때
    └── Wright 트윗 삭제 같은 노이즈에 임계값을 바꾸면 시스템 신뢰도 붕괴
```

## 8. 설계 체크리스트

```
임계값을 설계하거나 재조정할 때 반드시 확인할 7개 항목:

☐ 1. 이 임계값의 원천(Source)이 명확한가?
      → 구조적 / 통계적 / 서사적 중 어디서 왔는지 기록

☐ 2. 3중 밴드(Watch-Alert-Hard)가 설정되어 있는가?
      → 단일 임계값은 all-or-nothing 함정

☐ 3. 지속 조건(일수)이 포함되어 있는가?
      → "순간 돌파"와 "지속 돌파" 구분

☐ 4. 절대값과 상대값이 병렬로 설계되어 있는가?
      → Hard KC는 "A OR B, 먼저 도달하는 것" 구조

☐ 5. 대응하는 Trigger와 페어링되어 있는가?
      → 페어 없는 KC = 도로에 안 올라탔는데 설치된 출구

☐ 6. KC 작동 시 행동이 사전 정의되어 있는가?
      → "그때 가서 판단"은 KC 가치의 90%를 소멸시킴

☐ 7. 재조정 주기와 조건이 명시되어 있는가?
      → 정기(월/분기/반기) + 비정기(판 전환 이벤트) 모두 설정
```
