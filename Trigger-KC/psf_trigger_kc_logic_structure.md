# PSF Trigger-KC 작동 논리 구조

> **핵심 정의**: Trigger는 "이 문을 열어라", KC는 "이 문이 잘못된 문이었다"
> KC의 진짜 가치는 울리는 순간이 아니라, **매일 안 울림으로써 "아직 맞다"를 확인해주는 침묵**에 있다.

---

## 1. Trigger와 Kill Condition의 본질적 구분

```
KC는 "멈추는 조건", Trigger는 "움직이는 조건"
│
├── Kill Condition (KC)
│   ├── 질문: "이 투자 논리가 틀렸음을 증명하는 것은 무엇인가?"
│   ├── 성격: 방어적 — thesis를 무효화하는 임계점
│   ├── 작동: 트리거되면 → 포지션 재검토 or 청산
│   └── 비유: 연기 감지기 — 울리면 대피
│
└── Trigger (Link-Trigger)
    ├── 질문: "이 전이 경로가 활성화되었음을 확인하는 것은 무엇인가?"
    ├── 성격: 탐색적 — 인과체인의 다음 노드 점화 확인
    ├── 작동: 트리거되면 → 시나리오 분기 진입 or 포지션 진입
    └── 비유: 신호등 — 바뀌면 출발
```

---

## 2. 완전한 PSF 작동 체인 (6단계)

```
Phase 0: 시나리오 설계
│   "어떤 미래가 가능한가?"
│   ├── 복수 시나리오 수립 (A/B/C)
│   ├── 각 시나리오에 Trigger 지표 배정
│   ├── 각 시나리오에 KC 지표 배정
│   └── 이 단계에서는 아무 행동 없음 — 순수 설계
│
Phase 1: 임계값 접근 감지
│   "숫자가 움직이기 시작했다"
│   ├── Trigger 지표가 임계값을 향해 이동 중
│   ├── 아직 임계값 미도달 — 관찰만
│   └── 여기서 하는 일: 접근 속도와 방향 측정
│
Phase 2: 임계값 충족 → Trigger 점화
│   "문이 열렸다"
│   ├── 지표가 사전 설정 임계값 도달/돌파
│   ├── 복수 Trigger 점화 밀도 확인 (1개? 2개? 전부?)
│   └── 점화 밀도가 충분하면 → 포지션 진입
│
Phase 3: 보유 + KC 능동 확인
│   "탔다, 매일 경보를 확인한다"
│   ├── KC가 안 울림 → thesis 유효 → 보유 지속
│   ├── KC가 울림 → Phase 4로 강제 이동
│   └── 핵심: "관망"이 아니라 "능동적 생존 확인"
│       ├── 관망 마인드셋: "KC가 울리면 그때 반응하자" → 반응 지연
│       └── 능동 확인 마인드셋: "오늘도 안 울렸다 = 오늘도 맞다" → thesis 확신도 갱신
│
Phase 4: KC 작동 → thesis 재검토
│   "경보가 울렸다"
│   ├── 선택지 1: 포지션 청산 (thesis 포기)
│   ├── 선택지 2: 포지션 축소 (부분 인정)
│   └── 선택지 3: 시나리오 전환 (A→B로 갈아탐)
│
Phase 5: 목표 도달 → 정상 이탈
    "맞았다, 수확한다"
    ├── KC가 한 번도 안 울린 채 목표가 도달
    └── 이것이 이상적 완주
```

**한 문장 요약**: 시나리오를 설계하고(Phase 0), 임계값이 충족되면 Trigger가 문을 열고(Phase 2), 그 이후부터는 KC가 "아직 맞다"를 매일 확인해주는(Phase 3) 것이 PSF의 의사결정 엔진이다.

## 3. Trigger-KC 4가지 조합 상태

```
┌─────────────────┬──────────────────────────┬──────────────────────────┐
│                  │ KC 미작동                │ KC 작동                  │
├─────────────────┼──────────────────────────┼──────────────────────────┤
│ Trigger 점화    │ 🟢 최적                   │ 🔴 thesis 붕괴            │
│                 │ "타라, 그리고 아직 맞다"  │ "탔는데 틀렸다, 내려라"  │
│                 │ → 확신도 높은 보유        │ → 손절/시나리오 전환     │
├─────────────────┼──────────────────────────┼──────────────────────────┤
│ Trigger 미점화  │ ⚪ 대기                    │ 🟡 구조 경고(판 전환)     │
│                 │ "아직 아무것도 아님"      │ "타지도 않았는데          │
│                 │ → 관찰 지속              │  판이 바뀌고 있다"        │
│                 │                          │ → 전체 프레임 재검토     │
└─────────────────┴──────────────────────────┴──────────────────────────┘
```

**가장 위험한 상태**: 🟡 Trigger 미점화 + KC 작동 — 포지션이 없는데 KC가 울린다는 것은 시장 구조(regime) 자체가 전환 중이라는 의미. PSF에서 가장 경계하는 "판 전환(regime change)" 시그널.

## 4. 설계 규율: Trigger-KC 페어링

```
원칙: 모든 Trigger에는 대응하는 KC가 있어야 한다

├── Trigger: "호르무즈 부분 재개 뉴스" → de-escalation 진입
│   └── 대응 KC: "72시간 내 이란 재공격" → de-escalation 무효
│
├── Trigger: "나프타/Brent 괴리 사상 최고" → 석유화학 충격 진입
│   └── 대응 KC: "나프타/Brent 비율 0.85x 하회" → 공급 정상화
│
├── Trigger: "BEI 5Y5Y 2.8% 돌파" → 인플레 경로 진입
│   └── 대응 KC: "BEI 2.2% 하회" → 인플레 기대 꺾임
│
└── 설계 결함 패턴
    ├── 페어 없는 Trigger = 출구 없는 고속도로
    ├── 페어 없는 KC = 도로에 안 올라탔는데 설치된 출구
    └── 둘 다 PSF 시스템의 방어 기능 무력화
```

## 5. Phase별 감시 대상 전환 규칙

```
Phase가 바뀌면 감시 대상도 바뀐다 — 동시에 같은 무게로 보면 의사결정 마비

├── Phase 0~1: Trigger 지표만 감시
│   ├── "임계값까지 얼마나 남았나?"
│   ├── KC는 설계만 해두고 아직 작동시키지 않음
│   └── 이 단계에서 KC를 보면: 판단 근거 없이 불안만 증폭
│
├── Phase 2 (전환점): Trigger → KC로 감시 주체 교대
│   ├── Trigger 점화 확인 → 포지션 진입
│   ├── 이 순간부터 Trigger는 더 이상 중요하지 않음 (이미 탔으니까)
│   └── KC 감시 시작 — "매일 체크리스트"
│
├── Phase 3: KC만 감시
│   ├── "오늘 경보 안 울렸다" = thesis 유효 기간 1일 연장
│   ├── Trigger를 다시 보면: 이미 결정한 것을 다시 의심 → 확증편향 or 불필요한 재검토
│   └── 예외: 새로운 시나리오의 Trigger는 별도 트랙으로 감시 가능
│
└── Phase 4~5: 이탈 판단
    ├── KC 작동 → 즉시 재검토 (지체 없이)
    └── 목표 도달 → 정상 이탈 (탐욕 방지)
```

## 6. 실전 적용 사례: 이란 전쟁 유가 (2026.03.26 기준)

### 시나리오 A: de-escalation

```
[Trigger 패널]                    [KC 패널]
┌──────────────────────┐         ┌──────────────────────┐
│ T-1: Brent < $100    │──페어──→│ KC-1: 이란 재공격     │
│     ✅ 점화           │         │   (72h 내 에너지 타격)│
│     3/25 $98 도달     │         │     ❌ 미작동          │
├──────────────────────┤         ├──────────────────────┤
│ T-2: 호르무즈 부분재개│──페어──→│ KC-2: 통항 재차단     │
│     ❌ 미점화         │         │   (통항량 재하락)     │
│     통항 -80% 유지    │         │     ❌ 미작동          │
└──────────────────────┘         └──────────────────────┘

상태: Trigger 1/2 점화 | KC 0/2 미작동
판정: 🟡 반쪽 진입 — 탐색적 소규모 포지션만 허용
다음 확인: T-2 점화 여부 (호르무즈 통항 재개 뉴스)
```

### 시나리오 B: 스태그플레이션

```
[Trigger 패널]                    [KC 패널]
┌──────────────────────┐         ┌──────────────────────┐
│ T-3: 나프타/Brent     │──페어──→│ KC-3: 나프타/Brent    │
│   괴리 사상최고       │         │   0.85x 하회          │
│     ✅ 점화           │         │     ❌ 미작동          │
│     현재 1.04x        │         │                      │
├──────────────────────┤         ├──────────────────────┤
│ T-4: BEI 5Y5Y        │──페어──→│ KC-4: BEI 5Y5Y       │
│   2.8% 돌파           │         │   2.2% 하회           │
│     ✅ 점화           │         │     ❌ 미작동          │
└──────────────────────┘         └──────────────────────┘

상태: Trigger 2/2 점화 | KC 0/2 미작동
판정: 🟢 최적 조합 — 확신도 높은 포지션 (인플레 헤지: 금, TIPS, 원자재)
다음 확인: KC-3, KC-4 매일 능동 확인
```

### 두 시나리오 교차 판독

```
├── de-escalation: Trigger 불완전(1/2) → 아직 확신 불가
├── 스태그플레이션: Trigger 완비(2/2) + KC 클리어(0/2) → thesis 유효
│
└── 괴리: 시장은 de-escalation을 프라이싱(Brent<$100)하지만,
    PSF 대시보드는 스태그 시나리오가 더 강하게 확인됨
    → 이 괴리 자체가 투자 기회 or 리스크의 원천
```

## 7. Trigger-KC 설계 체크리스트

```
시나리오 설계 시 반드시 확인할 5개 항목:

☐ 1. 모든 Trigger에 대응 KC가 페어링되어 있는가?
      → 없으면 출구 없는 포지션

☐ 2. KC의 임계값이 사전에 정량적으로 정의되어 있는가?
      → "상황이 나빠지면"은 KC가 아님. "$150 3일 지속"이 KC

☐ 3. Trigger 점화 밀도 기준이 있는가?
      → 3개 중 1개 점화 vs 3개 중 3개 점화는 확신도가 다름

☐ 4. Phase별 감시 대상이 명확히 분리되어 있는가?
      → Phase 1에서 KC를 보면 판단 마비, Phase 3에서 Trigger를 보면 재의심

☐ 5. KC 작동 시 대응 행동이 사전에 정의되어 있는가?
      → "그때 가서 판단하자"는 KC의 가치를 90% 소멸시킴
      → "KC-1 작동 → 포지션 50% 축소 + 시나리오 B로 전환" 수준으로 구체적
```

## 8. 비유 종합

```
도로 시스템
├── 시나리오 = 가능한 경로들이 표시된 지도
├── 임계값 접근 = 고속도로 진입 램프가 보이기 시작
├── Trigger = 램프 신호등이 초록불 → "올라타라"
├── KC = 도로 위 비상 출구 → "여기서 내려라"
├── KC 미작동 = "이 도로 맞다, 계속 가라"
└── 목표 도달 = 목적지 도착 → 정상 하차

건강검진
├── Trigger = "이 증상이 있으니 정밀검사 받아라" (진입 조건)
├── KC = "이 수치가 나오면 진단이 바뀐다" (무효화 조건)
├── 증상 있음 + 정상 수치 = 🟢 "안심하고 치료 진행"
└── 증상 있음 + 이상 수치 = 🔴 "치료 방향 전면 변경"
```
