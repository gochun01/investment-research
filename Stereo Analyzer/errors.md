# Stereo Analyzer — 오류 기록 + 셀프검증 체크리스트

> 오류는 한 번의 실수. 반복은 시스템의 결함.
> 모든 분석 완료 후 이 체크리스트를 실행한다.

## 셀프검증 체크리스트 (12항목)

분석 완료 후 아래 항목으로 자가 검증. 하나라도 No이면 보강 후 출력.

### Phase 0: Pre-Read 검증

- [ ] **Phase 0 Gate를 MCP 호출보다 먼저 실행했는가** (P-01 방어: history/+tracking/ 스캔 → 맥락 출력 → 그 다음 수집)
- [ ] Pre-Read(Type/SCP/Urgency) 판독을 실행했는가
- [ ] 과거 맥락 로딩(0-5)을 실행했는가 (tracking cards + history 검색. 있으면 델타 모드.)
- [ ] SCP를 과대/과소 판독하지 않았는가 (수식어 제거 후 재판독)
- [ ] NOISE 판정 시 L1만으로 종료했는가 (불필요한 분석 방지)
- [ ] 감정 분리가 필요한데 건너뛰지 않았는가

### Phase 2: 7-Layer 검증

- [ ] L2에서 "기사가 말하지 않은 것" 2개+ 찾았는가
- [ ] L2 빈칸 중 MCP로 즉시 답 가능+핵심인 것을 Phase 2.3에서 수집했는가 (L-05 방어: "아는데 안 한 것" 방지)
- [ ] L2 팩트가 전부 🟢인가? → 전부 🟢이면 🟡/🔴 후보를 재점검 (L-02 방어)
- [ ] L4에서 표면적 원인과 구조적 원인을 분리했는가
- [ ] L5에서 반복/전환/모호 판정에 근거를 달았는가
- [ ] L6에서 second-order-patterns.md의 패턴을 1개+ 적용했는가 (L-01 방어)
- [ ] L6에서 기사 본문에 없는 새로운 연결을 도출했는가
- [ ] L6에서 한국 노출이 있으면 국내 전이 경로를 구체적으로 분석했는가 (L-03 방어)
- [ ] 핵심 수치가 단일 출처면 ⚠ 표시 + 교차검증 소스를 명시했는가 (L-04 방어)
- [ ] L7에서 노이즈 vs 시그널 조건이 검증 가능한가

### Phase 3~4: 돌발 질문 + 불확실성 검증

- [ ] 돌발 질문을 최소 1개 생성했는가
- [ ] 불확실성 지도를 작성했는가

### 코드↔스키마 정합성 (O-01 방어)

- [ ] render_adaptive.py 수정 시 SCHEMAS.md의 키 이름과 대조했는가
- [ ] SCHEMAS.md 키 변경 시 render_adaptive.py의 해당 키도 함께 변경했는가

### Phase 2.5: 되먹임 검증

- [ ] POLICY 또는 STRUCT 유형이면 FB-2(L5→L3)를 실행했는가 (F-01 방어)
- [ ] FB-4 최종 대조를 실행했는가 ("처음과 다르게 보이는가?")
- [ ] SCP ≥ 3이면 FB-5(시간축 검증)를 실행했는가 (R-07 방어: 스냅샷→구조 점프 방지)
- [ ] SCP ≥ 2이면 L7 시나리오에 Trigger-KC 페어가 부착되어 있는가 (페어 없는 시나리오 = 출구 없는 고속도로)
- [ ] Trigger-KC 설계 7항목 체크리스트를 통과했는가 (CLAUDE.md "Trigger-KC 설계 규율" 참조: 원천·3중밴드·지속조건·절대+상대·페어링·사전행동·재조정)

## 인지 오류 방어 카탈로그 (R-01 ~ R-06)

분석 중 자동으로 점검해야 할 인지 오류 패턴.

### R-01: 프레이밍 오염

```
증상: 기사의 프레이밍이 분석 전체를 지배한다.
      기사가 "위기"라고 쓰면 분석도 "위기" 전제로 진행.
탐지: L1 프레이밍 태그와 L4~L7 결론의 톤이 일치하면 의심.
      "내 결론이 기사의 프레이밍과 왜 같은가?" 자문.
방어: L1에서 프레이밍을 먼저 식별한 후, L4부터는 프레이밍을 벗기고 분석.
      "프레이밍을 뒤집으면 분석이 달라지는가?" 테스트.
심각도: 높음 — 전체 분석을 오염시킴
```

### R-02: SCP 과대 판독

```
증상: "최초", "사상 최대", "전례 없는" 같은 수식어에 끌려 SCP를 높게 잡는다.
      한 기업의 이벤트를 산업 구조 변화로 확대 해석.
탐지: SCP 3 이상을 부여했을 때, 수식어를 모두 제거하고 팩트만 놓고 재판독.
      "이것이 다른 기업에도 적용되는가?" 자문.
방어: autonomous-thinking-guide.md §1의 SCP 함정 방지 규칙 적용.
      L1 프레이밍 태그 식별 → SCP 판독 순서 준수.
심각도: 중간 — 분석 깊이 배분을 왜곡
```

### R-03: 균일 깊이 (v1.0 퇴행)

```
증상: 모든 Layer를 같은 분량으로 채운다. 빈칸 채우기 모드.
      Pre-Read의 라우팅 지시를 무시하고 "전부 다 열심히".
탐지: L1~L7 분량이 비슷하면 의심.
      "Pre-Read에서 집중하라고 한 Layer가 실제로 가장 깊은가?" 확인.
방어: Pre-Read 라우팅 결과를 출력 상단에 명시.
      집중 Layer는 상세하게, 축소 Layer는 1~2줄로.
심각도: 중간 — v2.0의 핵심 원칙 위반
```

### R-04: 확증 편향

```
증상: L4에서 첫 번째 인과를 찾으면 대안 탐색을 중단한다.
      "왜 지금"에 대한 답이 하나만 있다고 가정.
탐지: L4 인과 트리에 가지가 1개뿐이면 의심.
      "이것 말고 다른 설명은 없는가?" 자문.
방어: 표면적 원인 vs 구조적 원인 분리를 강제.
      최소 2개의 인과 가지를 탐색한 후 가장 설득력 있는 것을 선택.
심각도: 높음 — 인과 분석의 근본을 훼손
```

### R-05: 돌발 질문 회피

```
증상: 편한 질문만 생성한다. 기사의 프레이밍 안에서 나올 수 있는 질문.
      "이미 답을 아는 질문"을 돌발 질문이라고 포장.
탐지: 돌발 질문의 답이 기사 본문에 이미 있으면 의심.
      "이 질문의 답을 모르는가?" 자문. 안다면 돌발이 아님.
방어: autonomous-thinking-guide.md §3의 "좋은 질문의 조건" 대조.
      답을 모르지만, 답을 알면 판단이 극적으로 달라지는 질문인지 확인.
심각도: 중간 — Phase 3의 가치를 소거
```

### R-06: 감정 동조

```
증상: 사용자의 감정(공포/흥분)에 분석이 끌려간다.
      "망하는 거 아냐?"에 대해 위기 시나리오를 과도하게 전개.
      "대박 가즈아!"에 대해 상승 시나리오에 편중.
탐지: 감정 분리를 수행했으나 L4~L7이 여전히 감정의 방향으로 편향.
      "감정 반대 방향의 시나리오를 얼마나 다뤘는가?" 확인.
방어: 감정 분리 3단계를 충실히 실행.
      변환된 질문이 중립적인지 확인.
      L7 회답에서 양방향 조건 (실현 조건 + 불발 조건)을 모두 제시.
심각도: 높음 — 분석의 객관성 훼손
```

### L-05: 빈칸 발견→수집 미실행

```
증상: L2에서 "기사가 말하지 않은 것"을 정확히 식별했으나,
      MCP로 즉시 답할 수 있는데도 수집하지 않고 돌발 질문으로 미룸.
      사용자가 "수집해줘"를 해야 비로소 행동.
탐지: L2 "말하지 않은 것" 목록과 Phase 3 돌발 질문을 대조.
      "WebSearch 1회로 답 가능한 것이 돌발 질문에 있으면" 의심.
방어: Phase 2.3(빈칸 수집) 실행.
      L2 빈칸 각각에 "MCP로 즉시 답 가능 + 분석 핵심?" 판단.
      YES면 즉시 수집. NO면 돌발 질문으로 이관.
심각도: 높음 — 핵심 데이터가 보고서에서 빠짐. 사용자 신뢰 저하.
발견: 2026-03-26 SA-004 (ISA/ETF). 해외 ETF 53.2%가 보고서에 누락.
```

### R-07: 스냅샷→구조 점프

```
증상: 오늘 하루 데이터의 괴리를 "구조적 전환"이라고 단정한다.
      "DXY↓인데 KRW↓ = 시스템 사각지대"처럼
      스냅샷 괴리를 시스템 결함으로 점프.
탐지: "구조적"이라는 단어를 쓸 때, 그 근거가 3개월+ 시계열인가 확인.
      L5에서 시간축 분해를 했는가? 안 했으면 의심.
      "이 판단을 오늘 하루 데이터만으로 내려도 되는가?" 자문.
방어: L5 시간축 분해 (SCP ≥ 3일 때 필수).
      FB-5 시간축 검증 실행.
      스냅샷 기반이면 "현재 구간에서"로 한정. "구조적"은 3개월+ 근거 필요.
심각도: 높음 — 과장된 결론이 투자 판단에 직접 영향
발견: 2026-03-26 SA-20260326-002 (KRW 입체 해부)
```

## 오류 기록 규칙

```
1. 오류를 발견하면 즉시 위 테이블에 기록한다.
2. "재발 방지"에서 체크리스트 항목을 추출하여 상단에 추가한다.
3. 해결된 오류도 삭제하지 않는다. 패턴이 보이려면 이력이 필요하다.
4. 같은 오류 코드가 3회 이상 반복되면 SKILL.md 또는 가이드 수정을 검토한다.
5. 오류 코드 체계:
   - R-XX: 인지 오류 (위 카탈로그)
   - P-XX: Pre-Read 오류
   - L-XX: Layer 분석 오류
   - F-XX: 되먹임 오류
   - O-XX: 출력/렌더링 오류
   - D-XX: 데이터 기반 보정 (학습 피드백 루프에서 생성)
```

## 데이터 기반 보정 규칙 (학습 피드백 루프)

> 이 섹션은 prediction-ledger.json의 적중률 데이터에서 자동 생성된다.
> 월 1회 Mode E(재보정) 실행 시 prediction 집계 → 오판 패턴 식별 → 여기에 규칙 추가.
> 수동 인지 오류(R-XX)와 달리, D-XX는 데이터가 근거. 적중률 수치를 반드시 명시.
> 최소 예측 10건 이상 축적 후 의미 있는 패턴 산출.

```
현재: 예측 8건 등록, 0건 resolved. 보정 규칙 생성 전.
```
