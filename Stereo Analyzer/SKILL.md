# Stereo Analyzer v2.0 (입체 분석 — 자율 사고 엔진)

## 핵심 철학

> **"빈칸을 채우는 것은 분석이 아니다. 어디에 힘을 줄지 스스로 결정하는 것이 사고다."**

v1.0은 L1~L7을 순서대로 채우는 "해부 매뉴얼"이었다.
v2.0은 입력을 먼저 판독하고, 문맥에 따라 분석 전략을 자율적으로 선택하며,
Layer 사이를 되먹임으로 오가고, 템플릿 밖의 질문을 스스로 생성하는 "사고 엔진"이다.

작동 원칙 3가지:
1. **판독 우선**: 분석 전에 이슈의 종류·무게·긴급도를 먼저 읽는다
2. **선택적 깊이**: 모든 Layer를 균일하게 채우지 않는다. 문맥이 요구하는 곳에 집중한다
3. **자기 인식**: 내가 확신하는 곳과 모르는 곳을 구분하여 표시한다

---

## 입력 형태

사용자는 다음 중 하나 이상을 제공한다:
- 기사 URL (단일 또는 복수)
- 기사 제목 + 본문 텍스트
- 쟁점/이슈 키워드 (예: "미중 AI칩 규제", "한전 적자전환")
- 걱정/질문 형태 (예: "반도체 수출 괜찮을까?")

없으면 사용자에게 하나라도 던져달라고 요청한다.

## Phase 0: Pre-Read & Context Routing (분석 전 자율 판독)

**이 Phase가 전체 분석의 방향을 결정한다. 생략 금지.**

### 0-1. 이슈 유형 분류 (Type)

입력을 읽고 아래 6가지 중 하나(또는 복합)로 분류한다:

| 코드 | 유형 | 특징 | 분석 중심 Layer |
|------|------|------|----------------|
| `POLICY` | 정책·규제·법률 변화 | 정부/기관 발표, 입법 동향 | L4(왜 지금)+L6(2차효과) |
| `MACRO` | 매크로/금리/유동성/지정학 | 글로벌 변수, 시장 전체 영향 | L4+L5(구조)+L6 |
| `STRUCT` | 산업 구조·밸류체인 변화 | 기술전환, M&A, 공급망 재편 | L5(구조)+L6+L7 |
| `EVENT` | 실적·인사·단발 이벤트 | 특정 기업/인물 중심 | L2(팩트)+L3(이해관계)+L7 |
| `NARR` | 시장 심리·내러티브 전환 | 컨센서스 변화, 공포/탐욕 전환 | L1(프레이밍)+L4+L5 |
| `NOISE` | 반복 보도·재탕·감정 소비형 | 새 팩트 없음, 프레이밍만 다름 | L1만 → 노이즈 선언 후 종료 |

**복합 분류 가능**: 예) `POLICY×STRUCT` = 규제가 산업 구조를 바꾸는 경우

### 0-2. 구조 변화 잠재력 (SCP: Structural Change Potential)

이 이슈가 기존 구조를 얼마나 흔들 수 있는지 0~5로 판독한다:

```
SCP 0  — 구조와 무관한 노이즈
SCP 1  — 구조 내 일상적 변동 (실적 비트/미스, 인사이동)
SCP 2  — 구조 내 중요 변수의 변화 (점유율 이동, 마진 변화)
SCP 3  — 구조 변화의 초기 신호 (신규 진입자, 기술 전환 징후)
SCP 4  — 구조 재편 진행 중 (밸류체인 이동, 규제 패러다임 전환)
SCP 5  — 구조 파괴/창조 (기존 시스템 무효화, 새 시스템 등장)
```

**SCP 판독 근거를 반드시 1문장으로 명시한다.**
예) `SCP 3 — "미국 AI칩 수출규제가 중국 자체 GPU 생태계를 가속시키는 초기 징후"`

### 0-3. 시간 긴급도 (Urgency)

| 긴급도 | 기준 | 분석 속도 |
|--------|------|----------|
| `URGENT` | 24~72시간 내 판단/행동 필요 | 빠른 모드 우선, 핵심만 |
| `WATCH` | 1~4주 내 추적 필요 | 풀 모드, 추적 지표 중시 |
| `SLOW` | 장기 관찰 대상 | 구조 분석 중시, 판단 유보 가능 |

### 0-4. 감정 분리 (Emotion Decoupling) — 해당 시 자동 발동

사용자 입력에 감정이 실려 있으면(걱정, 불안, 흥분, 분노 등) **분석 전에 감정을 먼저 분리한다.**

감정 신호 탐지 키워드: "걱정", "불안", "괜찮을까", "망하는 거 아냐", "어떡하지",
"대박", "미쳤다", "말이 되냐", "끝났다", "폭락", "폭등", "ㅋㅋ", "ㄷㄷ" 등

감정 분리 3단계:
```
Step 1. 감정 인정
  "이 걱정의 핵심은 ___에 대한 불확실성이다"
  (감정을 부정하거나 무시하지 않는다)

Step 2. 검증 가능한 질문으로 변환
  걱정: "반도체 수출 망하는 거 아냐?"
  → 변환: "한국 반도체 수출의 구조적 경쟁력이 훼손되는 조건은 무엇인가?"
  (감정의 프레임을 벗기고, 검증 가능한 형태로 재구성)

Step 3. 변환된 질문으로 L1~L7 진행
  분석 완료 후 L7에서 원래 걱정에 대해 명시적으로 회답:
  "당신의 걱정이 현실화되려면 ___가 ___해야 한다.
   현재 그 조건은 ___이므로, [시그널/노이즈] 판정"
```

### 0-5. 과거 맥락 로딩 (Prior Context) ★

**새 분석을 "처음부터"가 아니라 "이전 분석 위에서" 시작한다.**

```
Step 1: tracking/cards/ 스캔
  새 이슈의 키워드가 기존 TC 카드에 매칭되는가?

  YES → 해당 카드 JSON + 원본 Stereo history JSON 로딩
    → Phase, Trigger/KC 상태, 마지막 check_log 확인
    → "이전 분석 존재. 델타(변화)에 집중."
  NO  → Step 2로

Step 2: Stereo history/*.json 검색
  새 이슈의 키워드로 history/ 파일명 + title 필드 검색
  매칭 1~2건 → 요약 로딩 (one_line, scenarios, key_insight, feedback)
  매칭 없음 → "첫 분석. 풀 모드."

Step 3: rm history/ 검색 (선택)
  같은 이슈의 rm 수집 결과가 history/에 있으면 참조
  5계층 패턴 판독 결과를 맥락으로 사용

산출:
  prior_context: {
    has_prior: true/false,
    tc_card: "TC-001" 또는 null,
    tc_phase: 1,
    prior_analysis: "SA-20260326-001: IEEPA→301 3단 진화. Phase 1.",
    prior_scenarios: {A: "25%", B: "35%", ...},
    delta_focus: "이전 대비 무엇이 바뀌었는가"
  }

과거 맥락이 있을 때 분석이 달라지는 점:
  L4: "왜 지금" → "이전 분석의 전제가 맞았는가"로 전환
  L5: 구조를 다시 발견하지 않고 "변화"만 추적
  L7: TC 카드의 Phase 전환 여부 판단 + Trigger/KC 갱신
  → 분석 품질 ↑ + 시간 ↓ + 연속성 ✅

과거 맥락이 없을 때:
  기존대로 풀 모드 실행. 변화 없음.

실행 비용: 파일 검색 2~3회. MCP 0회. 30초.
```

### 0-6. 분석 전략 결정 (Context Router)

Pre-Read 결과(Type + SCP + Urgency + 감정 여부 + **과거 맥락**)를 종합하여
**어떤 Layer에 가중치를 두고, 어떤 Layer를 축소/생략할지** 자율적으로 결정한다.

라우팅 규칙:

```
IF Type = NOISE (SCP 0~1)
  → L1만 출력 + "이것은 노이즈입니다. 이유: ___" 선언
  → 나머지 Layer 생략. 분석 종료.

IF Urgency = URGENT
  → 빠른 모드: L1 + L4 + L7 집중
  → L2·L3·L5·L6은 핵심 1줄씩만

IF SCP ≥ 4
  → L5(구조)+L6(2차효과)에 최대 가중치
  → "구조가 바뀌고 있다"는 전제 하에 분석

IF SCP ≤ 1 AND Type = EVENT
  → L2(팩트)+L3(이해관계)+L7(판단) 중심
  → L5·L6은 간략하게

IF 감정 분리가 발동됨
  → 감정 분리 결과를 출력 상단에 배치
  → L7에서 원래 감정에 대한 직접 회답 필수

IF 입력이 URL 3개 이상 또는 "오늘 뉴스" 스타일
  → 배치 모드: 전체를 L1+Pre-Read로 스캔 → SCP 최고 1개를 풀 분석

IF 동일 이슈에 대한 이전 Notion 분석이 존재
  → 비교 모드: 이전 분석과의 델타(변화)에 집중

IF prior_context.has_prior == true (0-5에서 과거 맥락 발견)
  → 델타 모드: L4는 "이전 전제가 맞았는가"에 집중.
    L5 구조는 이전 분석 재활용 + 변화만 추적.
    L7 시나리오는 이전 시나리오 대비 확률 갱신 + TC Phase 전환 판단.
    "처음부터"가 아니라 "이전 분석 위에서" 분석.

OTHERWISE
  → 풀 모드: L1~L7 전체, Type에 따른 가중치 적용
```

**라우팅 결과를 출력 상단에 명시한다:**
```
🧭 Pre-Read: [TYPE] SCP [N] [URGENCY]
📂 과거 맥락: [있음: SA-XXX + TC-XXX / 없음: 첫 분석]
📐 분석 전략: [어디에 집중하고 어디를 축소하는지 1문장]
```

## Phase 2: 7-Layer 해부 (가중치 적용 + 되먹임)

### ━━ 보이는 영역 (Surface) ━━

#### L1. 헤드라인 디코딩 (What They Say)
- 기사의 **문자 그대로의 주장**을 1~2문장으로 압축
- 헤드라인의 **프레이밍 기법** 식별 (상세: `references/framing-catalog.md`)
- 🏷️ 프레이밍 태그: `[공포]` `[낙관]` `[권위호소]` `[대비효과]` `[모호]` `[긴급]`

#### L2. 팩트 스켈레톤 (What Actually Happened)
- **검증 가능한 사실**만 추출 (숫자, 날짜, 당사자, 공식 발언)
- 각 팩트에 신뢰도: 🟢확인됨 🟡미확인 🔴충돌 ⚫출처없음
- **기사가 말하지 않은 것** 최소 2개 명시 ← L2의 핵심

#### L3. 이해관계 지도 (Who Benefits, Who Loses)
- 관련 **플레이어 맵** (이익/손해/숨은 동기)
- **기사 유통 주체**(언론사, 기고자, 인용 전문가)의 동기 포함

### Phase 2.3: 빈칸 수집 (L2 보강) ★

```
L2에서 식별한 "기사가 말하지 않은 것" 각각에 대해:

Q: "이것을 MCP(WebSearch/Tavily/FRED/Yahoo) 1~2회로 지금 답할 수 있는가?"

  YES + 분석에 핵심 → 즉시 수집. L2에 [보강] 태그로 추가.
    기준: 이 데이터가 L5(구조) 또는 L6(2차 효과)의 결론을 바꿀 수 있는가?
    바꿀 수 있으면 → 핵심. 즉시.
    ★ 추가 기준 (L-06 방어): L5 구조 판정의 비교 대상이 되는 선례 수치도 "핵심".
      예: "TSMC ADR 상장 후 PER 변화" → L5에서 선례 비교의 근거. 핵심으로 분류.
    예: "ISA 해외 ETF 비중" → WebSearch 1회. L6 KRW 전이에 직결.

  YES + 분석에 보조 → 선택적 수집.
    예: "ISA 가입자 연령 분포" → 있으면 좋지만 L5/L6 결론 불변. 생략 가능.
    ★ 주의: "선례 수치"를 "보조"로 분류하지 말 것. 선례는 L5의 핵심 근거.

  NO (데이터 미발표, 미래 결과, 주관적 판단) → Phase 3 돌발 질문으로 이관.
    예: "전쟁 중 ISA 유입 지속?" → 2~3월 미발표. 수집 불가. 돌발 질문.

MCP 호출: 최대 2회. 이것은 Phase 1(기사 수집)의 보충이지 새 스캔이 아님.
빈칸이 전부 NO이면 → Phase 2.3 건너뜀. 0회 호출.

발견 배경:
  2026-03-26 SA-004(ISA/ETF). L2에서 "해외 ETF 비중 미공개"를 식별했으나
  수집하지 않고 돌발 질문으로 미룸. 사용자가 직접 "수집해줘"를 해야 답 나옴.
  이것은 "아는데 안 한 것" = 시스템 결함. Phase 2.3으로 해소.
```

### ━━ 보이지 않는 영역 (Subsurface) ━━

#### L4. 인과 역추적 (Why Now, Why This Way)
- **"왜 지금 이 기사가 나왔는가"** — 가장 중요한 질문
- 인과체인 트리(├──/└──)로 시각화
- **표면적 원인** vs **구조적 원인** 반드시 분리

#### L5. 구조 레이어 (What System Is This Part Of)
- 상위 시스템 식별, 동심원 위치 매핑
- **반복(🔄) / 전환(🔀) / 모호(❓)** 판정 + 근거
- 역사적 전례 1~2개
- **(SCP ≥ 3일 때) 시간축 분해 (Temporal Decomposition)**:
  "현재 상태를 만든 힘을 시간 순서로 분해한다."
  "이것은 언제부터 시작되었고, 몇 단계를 거쳐 여기까지 왔는가?"
  ```
  ┌── [시기]: [원인] — 제거 시 [결과]
  ├── [시기]: [원인] — 제거 시 [결과]
  └── [시기]: [원인] — 제거 시 [결과]
  → 제거 시뮬레이션: 최근 충격 제거 시 [수준], 전부 제거 시 [수준]
  ```
  스냅샷과 추세를 구분하는 핵심 도구.
  scanner에서 [CUMULATIVE] 태그가 온 이슈는 반드시 실행.

#### L6. 2차 효과 체인 (What Happens Next That Nobody Is Talking About)
- 기사가 다루지 않는 **파생 효과** 도출 (상세 패턴: `references/second-order-patterns.md`)
- 시간 프레임: 단기/중기/장기
- 비선형 연결 (다른 분야로의 전이)
- 시나리오 분기 (if X then Y)
- **한국 노출이 있으면 국내 전이 경로를 구체적으로 분석** (L-03 방어:
  증권사별·연기금·보험사 노출 상세. "1줄 언급"으로 끝내지 않는다.)
- **핵심 수치가 단일 출처면 ⚠ 표시 + 교차검증 소스 명시** (L-04 방어:
  자극적 수치일수록 출처 검증이 중요. "46%"같은 수치는 2개+ 출처 필요.)

#### L7. 판단 프레임 (So What Do I Do With This)
- 투자 함의 + Kill Condition + 추적 지표
- **노이즈 vs 시그널** 판별 조건
- 감정 분리가 발동된 경우, 원래 걱정에 대한 직접 회답 포함

- **(SCP ≥ 2일 때) Trigger-KC 페어링**:
  L7에서 시나리오를 생성할 때, 각 시나리오에 Trigger-KC 페어를 반드시 부착한다.
  참조: `../Trigger-KC/psf_trigger_kc_logic_structure.md`
  참조: `../Trigger-KC/psf_threshold_design_protocol.md`

  ```
  각 시나리오에 포함할 것:

  Trigger (진입 조건):
    "이 시나리오가 현실화되었음을 확인하는 수치 조건"
    임계값 유형: 비율/스프레드 우선 (전이 감지에 최적)
    예: "Brent<$80 × 5d" / "BEI 5Y5Y 2.8% 돌파"

  KC (무효화 조건):
    "이 시나리오의 thesis가 틀렸음을 증명하는 수치 조건"
    3중 밴드: Watch → Alert → Hard KC
    지속 조건: × N일 (순간 돌파와 지속 돌파 구분)
    예: Watch $120 / Alert $135×3d / Hard $150×3d

  Phase 판정:
    현재 이 시나리오가 6-Phase 중 어디인가?
    Phase 0~1: Trigger 감시 중 (아직 미진입)
    Phase 2: Trigger 점화 (진입 시점)
    Phase 3: 보유 + KC 능동 확인 ("오늘도 안 울렸다 = 오늘도 맞다")
    Phase 4: KC 작동 (재검토)
    Phase 5: 목표 도달 (정상 이탈)

  출력 형식:
    시나리오 A (N%):
      Trigger: [조건] — Phase [현재]
      KC: Watch [W] / Alert [A] / Hard [H]
      현재: [수치] — [상태 ⚪🟡🔴]
  ```

  Trigger-KC 없는 시나리오는 "출구 없는 고속도로".
  SCP 0~1(노이즈/단발 이벤트)에서는 생략 가능.

## Phase 2.5: 되먹임 순회 (Feedback Loop)

L7까지 1차 완료 후, 아래 역방향 검증을 실행한다.
**기계적으로 전부 하지 않는다. Pre-Read의 SCP와 Type에 따라 가장 필요한 1~2개만 선택적으로 실행한다.**

```
되먹임 경로 4가지:

[FB-1] L4 완료 → L2 재점검
  "왜 지금"을 알고 나면, 기사가 생략한 팩트가 새로 보인다.
  → L2의 "기사가 말하지 않은 것"을 보강할 수 있는가?
  ★ POLICY, MACRO일 때 특히 중요

[FB-2] L5 완료 → L3 재점검
  구조를 알고 나면, 표면에 안 나온 숨은 플레이어가 보인다.
  → L3에 추가해야 할 이해관계자가 있는가?
  ★ STRUCT, POLICY일 때 필수 (F-01 방어: "권장"으로 두면 건너뜀.
     미실행 시 셀프체크 ❌ 처리.)

[FB-3] L6 완료 → L7 사전 조정
  2차 효과의 크기에 따라 판단 프레임의 긴급도가 달라진다.
  → L7의 KC나 추적 지표를 수정할 필요가 있는가?
  ★ SCP ≥ 3일 때 특히 중요

[FB-4] L7 완료 → L1 최종 대조 (항상 실행)
  "처음 읽었을 때와 지금 이 기사가 다르게 보이는가?"
  → 다르게 보이면: 분석 성공. 무엇이 달라졌는지 1문장으로 기록
  → 같으면: 분석 깊이 부족 경고. 가장 약한 Layer를 보강

[FB-5] L7 완료 → 시간축 검증 (SCP ≥ 3이면 실행)
  "이 결론을 오늘 하루 데이터만으로 내려도 되는가?"

  체크:
    □ L5에서 시간축 분해를 했는가?
    □ "구조적"이라는 단어를 썼다면, 근거가 3개월+ 데이터인가?
    □ 스냅샷 괴리를 구조적 판단으로 점프하지 않았는가?

  3개 YES → 통과
  NO가 있으면 → 해당 Layer 보강.
    "구조적" → "현재 구간에서" 등으로 수정.
    스냅샷 기반 판단에 [스냅샷 주의] 태그.

  ★ 이 FB는 "DXY↓+KRW↓=사각지대"류의 스냅샷→구조 점프를 방지한다.
    (2026-03-26 SA-002 오류에서 도출. errors.md 참조.)
```

되먹임으로 수정된 내용이 있으면 해당 Layer에 `[FB 보강]` 태그를 붙인다.

## Phase 3: 돌발 질문 생성 (Emergent Questions)

L4까지 분석한 시점에서, 정해진 7개 Layer의 질문 **밖에서**
이 이슈에 고유한 질문을 최소 1개, 최대 3개 생성한다.

생성 렌즈 5가지 (전부 쓰지 않는다. 이슈에 맞는 것만 선택):

```
[반전 렌즈]  "이것이 틀렸다면, 어떤 증거가 있어야 하는가?"
[시간 렌즈]  "이것이 6개월 전/후에 일어났다면 반응이 달랐을까?"
[침묵 렌즈]  "이 상황에서 가장 조용한 플레이어는 누구이며, 왜 조용한가?"
[규모 렌즈]  "이 숫자는 전체의 몇 %인가? 절대값인가 변화율인가?"
[역사 렌즈]  "과거에 이와 가장 구조적으로 비슷한 상황은? 그때 사람들은 뭘 놓쳤는가?"
```

돌발 질문은 L7 판단 프레임 바로 앞에 출력한다.
이 질문들에 대한 답까지 제시할 수 있으면 제시하고,
답할 수 없으면 "이것을 추적하면 시그널/노이즈 판별이 빨라진다"로 마무리한다.

## Phase 4: 불확실성 지도 (Uncertainty Map)

분석 완료 후, 각 Layer의 확신도를 자가 진단한다.

```
확신도 스케일:
■■■■■ 높음 — 검증된 팩트 기반, 출처 복수
■■■□□ 중간 — 합리적 추론이나 출처 단일 또는 간접
■□□□□ 낮음 — 가설 수준, 근거 약함
```

**가장 약한 고리**(확신도 최저 Layer)를 명시하고,
그것을 보강하려면 어떤 데이터/이벤트를 확인해야 하는지 제시한다.

```
⚠️ 약한 고리: L__ — ___
📡 보강 방법: ___를 확인하면 이 Layer의 확신도가 올라간다
```

## 자기 점검 규칙 (8항목)

분석 완료 후 아래 8개 질문으로 자가 검증.
하나라도 No이면 해당 부분을 보강한 후 출력한다.

1. **Pre-Read 체크**: Type·SCP·Urgency 판독 결과와 실제 분석 깊이가 일치하는가?
2. **L2 체크**: "기사가 말하지 않은 것"을 최소 2개 이상 찾았는가?
3. **L4 체크**: 표면적 원인과 구조적 원인을 분리했는가?
4. **L5 체크**: 반복/전환/모호 판정에 구체적 근거를 달았는가?
5. **L6 체크**: 2차 효과가 기사 본문에 없는 새로운 연결인가?
6. **L7 체크**: "노이즈 vs 시그널" 조건이 검증 가능한 형태인가?
7. **돌발 질문 체크**: 최소 1개의 Emergent Question을 생성했는가?
8. **FB-4 체크**: "처음과 다르게 보이는가?" 최종 대조를 했는가?

## 모드 변형

모드는 Context Router가 자동 선택하지만, 사용자가 직접 지정할 수도 있다.

| 트리거 | 모드 | 내용 |
|--------|------|------|
| 자동 또는 "빠르게 입체" | 🚀 빠른 | L1+L4+L7, 나머지 1줄 |
| 자동 또는 "기사 N개" | 📦 배치 | 전체 L1+Pre-Read → SCP 최고 1개 풀 분석 |
| 자동 또는 "판 연결" | 🔗 PSF연결 | L7에서 PSF Property 매핑 |

| 자동(Notion 기존 분석 존재) | 🔀 비교 | 이전 분석과의 델타에 집중 |
| 자동(SCP 0~1) | ⚡ 노이즈 차단 | L1만 → 노이즈 선언 → 종료 |
