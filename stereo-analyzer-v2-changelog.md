## 신규 추가 요소 상세

### 1. Phase 0: Pre-Read & Context Routing
| 요소 | 내용 | 왜 필요한가 |
|------|------|------------|
| Type 분류 (6가지) | POLICY/MACRO/STRUCT/EVENT/NARR/NOISE | 이슈 종류에 따라 어떤 Layer가 중요한지 결정 |
| SCP (0~5) | 구조 변화 잠재력 수치화 | SCP 높으면 L5·L6 집중, 낮으면 L2·L3·L7 중심 |
| Urgency | URGENT/WATCH/SLOW | 분석 속도와 깊이 조절 |
| 감정 분리 | 걱정→검증질문 변환 | "감정의 프레임 안에서 분석"하는 오류 방지 |
| Context Router | if-then 규칙으로 모드 자동 선택 | 사용자가 모드를 몰라도 최적 전략 적용 |

### 2. Phase 2.5: 되먹임 순회
| FB 경로 | 방향 | 언제 실행 |
|---------|------|----------|
| FB-1 | L4→L2 | POLICY, MACRO, NARR일 때 |
| FB-2 | L5→L3 | STRUCT, POLICY일 때 |
| FB-3 | L6→L7 | SCP ≥ 3일 때 |
| FB-4 | L7→L1 | **항상** (최종 대조) |

### 3. Phase 3: 돌발 질문 생성
| 렌즈 | 질문 패턴 | 우선 Type |
|------|----------|----------|
| 반전 | "틀렸다면 어떤 증거?" | POLICY, EVENT |
| 시간 | "6개월 전이라면 반응이 달랐을까?" | STRUCT, NARR |
| 침묵 | "가장 조용한 플레이어는?" | POLICY, STRUCT |
| 규모 | "전체의 몇 %인가?" | MACRO, EVENT |
| 역사 | "가장 비슷한 과거 상황은?" | MACRO, NARR |

### 4. Phase 4: 불확실성 지도
- Layer별 ■□ 5단계 확신도 시각화
- "가장 약한 고리" 명시
- 보강 방법(어떤 데이터를 확인해야 하는지) 제시
